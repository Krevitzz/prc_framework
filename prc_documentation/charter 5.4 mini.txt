## SECTION 1-3 : Fondations

### Axiomes (A1-A3)
- A1: Dissymétrie informationnelle D irréductible
- A2: Mécanisme Γ agissant sur D
- A3: Aucun Γ stable ne peut annuler D complètement

### Hiérarchie niveaux
```
L0=ontological → L1=epistemic → L2=theoretical → L3=operative → L4=documentary
Règle : L(n) ne référence que L(≤n-1)
```

### Core = 2 fonctions aveugles
```python
prepare_state(base, modifiers) → np.ndarray
run_kernel(state, gamma, ...) → Generator[iteration, state]
```

**Règles K1-K5** : Core aveugle (pas de validation, pas de sémantique, pas de classes)

## SECTION 11-12 : Pipeline opérationnel

### Workflow
```
1. Formulation HYP → 2. Spécification D/Γ → 3. Création TM
→ 4. Exécution kernel (db_raw) → 5. Tests+scoring (db_results)
→ 6. Discussion → 7. Mise à jour STATUS → 8. CON si pertinent
→ 9. Nouvelle HYP[R1] si pertinent
```

### Architecture batch_runner

**Principe** : 1 point d'entrée, 3 modes, chaînage auto

```bash
# Mode 1 : Collecte données
batch_runner --brut --gamma GAM-001
# → Exécute kernel, stocke db_raw (append-only, immuable)

# Mode 2 : Application tests
batch_runner --test --gamma GAM-001 --params default --scoring default
# → Vérifie db_raw, auto-lance --brut si manquant
# → Applique tests, calcule scores, stocke db_results

# Mode 3 : Génération verdicts
batch_runner --verdict --gamma GAM-001 --params default --scoring default --thresholds strict
# → Vérifie db_results, auto-lance --test si manquant
# → Calcule 3 critères, applique seuils, stocke verdict

# Raccourci
batch_runner --all --gamma GAM-001 --params default --scoring default --thresholds strict
```

**Règle non-reruns** : Toute modification scoring/verdict SANS réexécuter kernel

### Bases de données

**db_raw** (immuable, append-only) :
```sql
Executions   -- Métadonnées runs
Snapshots    -- États sauvegardés (compressed)
Metrics      -- Métriques brutes par iteration
```

**db_results** (rejouable, versionné) :
```sql
TestObservations  -- Résultats bruts tests
TestScores        -- Scores calculés (config_id)
GammaVerdicts     -- Verdicts agrégés (config_id + threshold_id)
```

## SECTION 12.4 : Tests - Format dict obligatoire

**Principe** : Tests OBSERVENT (pas de jugement), scoring INTERPRÈTE

### Status
- `SUCCESS` : Test exécuté normalement
- `ERROR` : Erreur technique → ARRÊT BATCH (bug code)
- `NOT_APPLICABLE` : Test invalide pour contexte (pas pénalisé)
- `SKIPPED` : Métrique spécifique omise (reste du test OK)

### Transitions factuelles (exemples)
```
"stable"      # Dans ±10% initial
"increasing"  # Croissance monotone
"explosive"   # Croissance > seuil
"converged"   # Distance successive < ε
"preserved"   # Propriété maintenue
"destroyed"   # Propriété perdue
```

### Interdictions formelles
```python
# ❌ JAMAIS dans retour test
'passed': bool
'failed': bool
'verdict': str
'score': float
'weight': float

# ❌ JAMAIS en code (YAML obligatoire)
if norm > 1000.0:  # Hardcodé
    ...

# ✅ TOUJOURS
params = load_test_parameters('UNIV-001')
if norm > params['explosion_threshold']:
    ...
```

### Architecture configs
```

  thresholds_strict_v1.yaml

config/tests/UNIV-001/      # Overrides spécifiques
```

### Verdict : 3 critères → 1 décision

**Critères** :
```python
majority_pct = (n_configs_pass / n_total_configs) * 100
robustness_pct = (n_D_with_viable_config / n_total_D) * 100
global_score = mean(all_config_scores)  # 0-1
```

**Logique (baseline)** :
```yaml
# SURVIVES[R0] : OU logique (1 critère suffit)
survives:
  score_min: 12.0      # /20
  robustness_min: 60.0 # %
  majority_min: 50.0   # %

# FLAGGED_FOR_REVIEW : ET logique (tous échouent)
flagged:
  score_max: 8.0
  robustness_max: 40.0
  majority_max: 20.0

# WIP[R0-closed] : Ni SURVIVES ni FLAGGED
```

**Verdicts** :
- `SURVIVES[R0]` : Non éliminé, viable pour exploration/composition
- `WIP[R0-closed]` : Ambigu, mise en attente (pas de promesse R1)
- `FLAGGED_FOR_REVIEW` : Échec probable, review manuelle obligatoire avant REJECTED

## SECTION 0 : Principes Directeurs

### 0.1. Philosophie

Le système à registres transforme le pipeline en un framework **extensible par conception**. Chaque registre est un **module autonome** encapsulant un domaine mathématique.

### 0.2. Règles Fondamentales

**R0-1** : Extensibilité sans modification du core  
**R0-2** : Découverte dynamique des registres au démarrage  
**R0-3** : Validation statique des signatures avant exécution  
**R0-4** : Documentation intégrée auto-générée  
**R0-5** : Aucun mode legacy - architecture unique

### 0.3. Terminologie Normalisée

| Terme | Définition |
|-------|------------|
| **run_metadata** | Métadonnées d'une exécution : {gamma_id, d_base_id, modifier_id, seed, state_shape} |
| **history** | Liste complète des snapshots d'un run (~200 états) |
| **snapshot** | UN état à une itération donnée (np.ndarray) |
| **registry** | Module contenant des fonctions mathématiques réutilisables |
| **registry_key** | Identifiant unique "registre.fonction" (ex: "algebra.matrix_norm") |
| **post_processor** | Transformation appliquée au résultat brut d'une métrique |

## SECTION 1 : Structure des Tests

### 1.1. Format Obligatoire
# prc_documentation/templates/test_xxx_yyy.py


### 1.2. Règles Strictes

**R1-1** : TEST_ID format "CAT-NNN" (3 lettres, tiret, 3 chiffres)  
**R1-2** : TEST_VERSION = "5.4" obligatoire  
**R1-3** : Docstring complète obligatoire (objectif + métriques + exclusions)  
**R1-4** : COMPUTATION_SPECS contient 1 à 5 métriques (pas 0, pas 10)  
**R1-5** : Chaque métrique a 'registry_key' et 'default_params' obligatoires  
**R1-6** : 'post_process' doit être une clé de POST_PROCESSORS (pas lambda string)  
**R1-7** : Aucun attribut FORMULAS (obsolète)

### 1.3. Validation

```python
# Validation automatique par discovery.py
def validate_test_structure(test_module):
    assert hasattr(test_module, 'TEST_ID')
    assert hasattr(test_module, 'TEST_VERSION')
    assert test_module.TEST_VERSION == "5.4"
    assert hasattr(test_module, 'APPLICABILITY_SPEC')
    assert hasattr(test_module, 'COMPUTATION_SPECS')
    assert not hasattr(test_module, 'FORMULAS')  # Obsolète
    assert 1 <= len(test_module.COMPUTATION_SPECS) <= 5
    
    for metric_name, spec in test_module.COMPUTATION_SPECS.items():
        assert 'registry_key' in spec
        assert 'default_params' in spec
        assert '.' in spec['registry_key']  # Format registre.fonction
```

## SECTION 7 : Workflow Complet

### 7.1. Pipeline

```
batch_runner --test --gamma GAM-001 --params default_v1
    ↓
[1] discovery.scan_tests()
    → {test_id: module} (COMPUTATION_SPECS obligatoire)
    ↓
[2] db.query("SELECT gamma_id, d_base_id, modifier_id, seed FROM Executions WHERE gamma_id=?")
    → Liste runs
    ↓
[3] Pour chaque run:
        run_metadata = {gamma_id, d_base_id, modifier_id, seed}
        ↓
        first_snapshot = db.load_first_snapshot(run_metadata)
        run_metadata['state_shape'] = first_snapshot.shape
        ↓
        Pour chaque test:
            ↓
            [4] applicability.check(test, run_metadata)
                ├─ Non → store NOT_APPLICABLE
                └─ Oui → continue
                    ↓
                    [5] history = db.load_all_snapshots(run_metadata)
                    ↓
                    [6] result = test_engine.execute_test(test, run_metadata, history, params_config_id)
                        ↓
                        RegistryManager.validate_computation_spec()
                        ↓
                        Pour chaque snapshot:
                            Pour chaque métrique:
                                function = registry.get_function(registry_key)
                                value = function(snapshot, **params)
                                post_process = POST_PROCESSORS[key]
                                final_value = post_process(value)
                                buffer.append(final_value)
                        ↓
                        Calculer statistics/evolution
                        ↓
                        return result
                    ↓
                    [7] store_result(result) → db_results
```

### 7.2. Optimisations

**O1** : Cache RegistryManager (fonctions)  
**O2** : Cache TestEngine (computations préparées)  
**O3** : Chargement state_shape = 1 snapshot uniquement  
**O4** : Chargement history = tous snapshots seulement si applicable

## SECTION 9 : Anti-Patterns

### 9.1. Tests

**AP1** : Trop de métriques
```python
# ❌ MAUVAIS (10 métriques)
COMPUTATION_SPECS = {
    'metric1': ...,
    'metric2': ...,
    # ... 10 au total
}

# ✅ BON (3 métriques bien choisies)
COMPUTATION_SPECS = {
    'metric1': ...,  # Principal
    'metric2': ...,  # Complémentaire
    'metric3': ...,  # Validation
}
```

**AP2** : Documentation absente
```python
# ❌ MAUVAIS
"""Test symétrie."""

# ✅ BON
"""
Évolution asymétrie matrices.

Objectif: ...
Métriques: ...
Exclusions: ...
"""
```

**AP3** : FORMULAS legacy
```python
# ❌ OBSOLÈTE
FORMULAS = {
    'metric': lambda state: np.linalg.norm(state),
}

# ✅ CORRECT
COMPUTATION_SPECS = {
    'metric': {
        'registry_key': 'algebra.matrix_norm',
        'default_params': {'norm_type': 'frobenius'},
    }
}
```

### 9.2. Registres

**AP4** : Fonction impure
```python
# ❌ MAUVAIS (modifie state)
def bad_function(state):
    state[0, 0] = 0
    return np.sum(state)

# ✅ BON (pure)
def good_function(state):
    return float(np.sum(state**2))
```

**AP5** : Docstring absente
```python
# ❌ MAUVAIS
def func(state, p):
    return state[0, 0]

# ✅ BON
def func(state, parameter=1.0):
    """
    Description complète.
    
    Args:
        state: Tenseur
        parameter: Description
    
    Returns:
        float: Valeur calculée
    """
    return float(state[0, 0] * parameter)
```

# SECTION 12.8 : SCORING - DÉTECTION PATHOLOGIES R0

## 12.8.1 Principe fondamental

En phase R0, **le scoring ne mesure pas des qualités, il détecte des pathologies structurelles**.

### Convention normalisée

```python
pathology_score ∈ [0, 1]
# 0 = aucun signal pathologique détecté (sain)
# 1 = pathologie maximale détectée (toxique)
```

**Affichage** : Les rapports peuvent afficher sur `/20` pour lisibilité (score × 20), mais **tous calculs internes utilisent [0,1]**.

### Interdictions formelles

- ❌ Interpréter un score élevé comme "bon"
- ❌ Optimiser pour maximiser un score
- ❌ Attribuer un "sens" positif aux métriques
- ❌ Compenser des pathologies critiques par moyenne

---

## 12.8.2 Typologie des pathologies (S1-S4 + MAPPING)

### TYPE S1 : COLLAPSE (seuil bas)

**Définition** : Valeur trop faible indique effondrement structural.

**Métriques typiques** :
- `entropy` : Perte diversité informationnelle
- `variance` : Collapse vers valeur unique
- `rank_estimate` : Perte dimensionnalité
- `spectral_gap` : Perte séparation modale

**Formule** :
```python
if mode == 'hard':
    score = 1.0 if value < threshold_low else 0.0
else:  # soft (default)
    score = max(0, min(1, (threshold_low - value) / threshold_low))

flag = value < critical_low
```

**Configuration YAML** :
```yaml
entropy:
  pathology_type: "S1_COLLAPSE"
  threshold_low: 0.1      # Début signal pathologique
  critical_low: 0.01      # Pathologie critique (flag)
  weight: 2.0
  mode: "soft"            # soft|hard
```

---

### TYPE S2 : EXPLOSION (seuil haut)

**Définition** : Valeur trop élevée indique divergence non contrôlée.

**Métriques typiques** :
- `spectral_radius` : Divergence itérative
- `condition_number` : Instabilité numérique
- `frobenius_norm` : Croissance explosive
- `gradient_magnitude` : Variations brutales

**Formule** :
```python
if mode == 'hard':
    score = 1.0 if value > threshold_high else 0.0
else:  # soft (default)
    score = max(0, min(1, (value - threshold_high) / threshold_high))

flag = value > critical_high
```

**Configuration YAML** :
```yaml
frobenius_norm:
  pathology_type: "S2_EXPLOSION"
  threshold_high: 1000.0
  critical_high: 10000.0
  weight: 2.0
  mode: "soft"
```

---

### TYPE S3 : PLATEAU (intervalle toxique)

**Définition** : Valeur trop stable ou uniformisée, perte structure exploitable.

**Métriques typiques** :
- `uniformity ≈ 1` : Distribution trop plate
- `sparsity ≈ 0` : Aucune parcimonie
- `symmetry_score ≈ 1` : Symétrie non désirée

**Formule** :
```python
interval = [interval_low, interval_high]

if interval[0] <= value <= interval[1]:
    score = 1.0
    flag = True
else:
    # Distance smooth à l'intervalle
    dist = min(abs(value - interval[0]), abs(value - interval[1]))
    score = max(0, 1 - dist)
    flag = False
```

**Configuration YAML** :
```yaml
uniformity:
  pathology_type: "S3_PLATEAU"
  interval_toxic: [0.9, 1.0]  # [min, max] toxique
  weight: 1.5
  mode: "soft"
```

---

### TYPE S4 : INSTABILITY (variation différentielle)

**Définition** : Changements trop rapides sous perturbation minime.

**Métriques typiques** :
- `Δentropy` : Volatilité entropique
- `Δrank` : Instabilité rang
- `volatility` : Variance des variations

**Formule** :
```python
# Nécessite historique ou contexte
delta = abs(value_t - value_t_prev)
score = max(0, min(1, delta / delta_critical))
flag = delta > delta_critical
```

**Configuration YAML** :
```yaml
evolution_volatility:
  pathology_type: "S4_INSTABILITY"
  delta_critical: 0.5
  weight: 1.0
  mode: "soft"
```

---

### TYPE MAPPING : Transitions qualitatives

**Définition** : Mapping direct de valeurs catégorielles vers scores pathologie.

**Métriques typiques** :
- `evolution_transition` : explosive, stable, converged...
- `evolution_trend` : increasing, decreasing, stable...

**Formule** :
```python
mapping = scoring_rule['mapping']
score = mapping.get(value, 0.5)  # Default 0.5 si non mappé
flag = score >= 0.8  # Seuil critique arbitraire
```

**Configuration YAML** :
```yaml
evolution_transition:
  pathology_type: "MAPPING"
  mapping:
    explosive: 1.0
    divergent: 1.0
    stable: 0.0
    converged: 0.0
    growing: 0.3
    shrinking: 0.3
    oscillating: 0.3
  weight: 3.0
```

## 12.8.4 Algorithme scoring

### Étape 1 : Scorer une métrique

```python
def score_metric(
    metric_key: str,
    metric_value: float | str,
    scoring_rule: dict,
    context: dict = None
) -> dict:
    """
    Calcule pathology_score pour une métrique.
    
    Args:
        metric_key: Nom métrique (ex: "stat_final")
        metric_value: Valeur observée
        scoring_rule: Config YAML règle scoring
        context: Contexte optionnel (historique, delta)
    
    Returns:
        {
            'metric_key': str,
            'value': float | str,
            'score': float [0,1],
            'flag': bool,
            'pathology_type': str,
            'weight': float,
            'evidence': dict
        }
    """
    pathology_type = scoring_rule['pathology_type']
    weight = scoring_rule.get('weight', 1.0)
    mode = scoring_rule.get('mode', 'soft')
    
    # TYPE S1 : COLLAPSE
    if pathology_type == "S1_COLLAPSE":
        threshold = scoring_rule['threshold_low']
        critical = scoring_rule.get('critical_low', threshold * 0.1)
        
        if mode == 'hard':
            score = 1.0 if metric_value < threshold else 0.0
        else:
            score = max(0, min(1, (threshold - metric_value) / threshold))
        
        flag = metric_value < critical
    
    # TYPE S2 : EXPLOSION
    elif pathology_type == "S2_EXPLOSION":
        threshold = scoring_rule['threshold_high']
        critical = scoring_rule.get('critical_high', threshold * 10)
        
        if mode == 'hard':
            score = 1.0 if metric_value > threshold else 0.0
        else:
            score = max(0, min(1, (metric_value - threshold) / threshold))
        
        flag = metric_value > critical
    
    # TYPE S3 : PLATEAU
    elif pathology_type == "S3_PLATEAU":
        interval = scoring_rule['interval_toxic']
        
        if interval[0] <= metric_value <= interval[1]:
            score = 1.0
            flag = True
        else:
            dist = min(abs(metric_value - interval[0]), 
                      abs(metric_value - interval[1]))
            score = max(0, 1 - dist)
            flag = False
    
    # TYPE S4 : INSTABILITY
    elif pathology_type == "S4_INSTABILITY":
        delta_critical = scoring_rule['delta_critical']
        
        if context and 'delta' in context:
            delta = context['delta']
        else:
            # Calcul delta depuis historique si disponible
            delta = 0  # Fallback
        
        score = max(0, min(1, delta / delta_critical))
        flag = delta > delta_critical
    
    # TYPE MAPPING
    elif pathology_type == "MAPPING":
        mapping = scoring_rule['mapping']
        score = mapping.get(metric_value, 0.5)  # Default 0.5 si non mappé
        flag = score >= 0.8
    
    else:
        raise ValueError(f"Unknown pathology_type: {pathology_type}")
    
    return {
        'metric_key': metric_key,
        'value': metric_value,
        'score': float(score),
        'flag': bool(flag),
        'pathology_type': pathology_type,
        'weight': weight,
        'evidence': {
            'threshold_config': scoring_rule,
            'context': context
        }
    }
```

### Étape 2 : Agréger métriques → test_score

```python
def aggregate_test_score(
    metric_scores: dict[str, dict],
    aggregation_mode: str = "max"
) -> dict:
    """
    Agrège scores métriques en score test.
    
    Args:
        metric_scores: {metric_key: score_dict}
        aggregation_mode: "max" | "weighted_mean" | "weighted_max"
    
    Returns:
        {
            'test_score': float [0,1],
            'pathology_flags': list[str],
            'critical_metrics': list[str],
            'aggregation_mode': str,
            'metric_scores': dict
        }
    """
    if not metric_scores:
        return {
            'test_score': 0.0,
            'pathology_flags': [],
            'critical_metrics': [],
            'aggregation_mode': aggregation_mode,
            'metric_scores': {}
        }
    
    # Agrégation selon mode
    if aggregation_mode == "max":
        # R0 default: une pathologie suffit
        test_score = max(m['score'] for m in metric_scores.values())
    
    elif aggregation_mode == "weighted_mean":
        # Future R1: pondération nuancée
        weighted_sum = sum(m['score'] * m['weight'] for m in metric_scores.values())
        weight_sum = sum(m['weight'] for m in metric_scores.values())
        test_score = weighted_sum / weight_sum if weight_sum > 0 else 0.0
    
    elif aggregation_mode == "weighted_max":
        # Hybride: max des scores pondérés
        weighted_scores = [m['score'] * m['weight'] for m in metric_scores.values()]
        test_score = max(weighted_scores) if weighted_scores else 0.0
    
    else:
        raise ValueError(f"Unknown aggregation_mode: {aggregation_mode}")
    
    # Flags et métriques critiques
    pathology_flags = [
        key for key, m in metric_scores.items() if m['flag']
    ]
    
    critical_metrics = [
        key for key, m in metric_scores.items() if m['score'] >= 0.8
    ]
    
    return {
        'test_score': float(test_score),
        'pathology_flags': pathology_flags,
        'critical_metrics': critical_metrics,
        'aggregation_mode': aggregation_mode,
        'metric_scores': metric_scores
    }
```

## 12.8.5 Clarifications normatives R0

### 12.8.5.1 Pathologies et pondération critique

**Principe général** :
> En R0, une pathologie systématique suffit à invalider la viabilité.

**Nuance contextuelle** :
Certaines pathologies peuvent être **pondérées comme critiques** via les `weight`, modifiant leur impact sur le score global. Cela ne contredit pas le principe car :

- La pathologie reste **détectée et tracée** (flag)
- Le poids reflète une **criticité contextuelle** (ex: symétrie pour mécanisme supposé préserver symétrie)
- En R0, nous tolérons des pathologies locales si le mécanisme reste **globalement ajustable**

**Exemple concret** :
```yaml
# Pour Γ censé préserver symétrie
asymmetry_norm:
  weight: 3.0  # Critique pour ce mécanisme

# Trace moins critique
trace_normalized:
  weight: 0.5
```

Un `test_score` élevé n'est pas un "rejet automatique" si d'autres tests compensent, **mais** :
- Les pathologies flaggées sont toujours **documentées**
- Le verdict explicite **pourquoi** cette pathologie n'a pas éliminé le candidat

---

### 12.8.5.2 Rôle du global_score : filtre, pas compensation

Le `global_score` n'est **pas une mesure de qualité globale**, mais un **indicateur de dispersion et concentration des pathologies**.

**Rôle exact** :
```python
global_score = mean(all_test_scores)  # [0,1]

# BUT : filtrer
# - Candidat avec UNE note critique isolée (global faible)
# - vs candidat avec pathologies systématiques partout (global élevé)
```

**Interprétation correcte** :
- `global_score` faible + 1 test critique → SURVIVES possible (pathologie locale, mécanisme ajustable)
- `global_score` élevé partout → REJECTED probable (pathologies systématiques)

**Ce que ce N'EST PAS** :
- ❌ Une compensation (mauvais partout ≠ bon globalement)
- ❌ Une moyenne de "qualités"
- ❌ Un score à optimiser

---

### 12.8.5.3 Conventions numériques : arbitraires par design

**Tous les seuils** (0.7, 0.8, 0.4, 0.6, 1000.0...) sont des **conventions initiales** pour exploration R0.

**Propriétés** :
- ❌ Aucune justification empirique
- ❌ Aucun fondement théorique
- ✅ Conventions de travail ajustables
- ✅ Baselines pour comparaison inter-configs

**Évolution attendue** :
```
R0 : Conventions larges (exploration, éviter faux négatifs)
R1 : Affinement empirique (patterns observés)
R2+: Calibration cross-domaines
```

Ces valeurs **doivent** être testées via configs alternatives (strict, lenient) pour évaluer sensibilité des verdicts.

# SECTION 12.9 : VERDICTS - AGRÉGATION GLOBALE R0

## 12.9.1 Principe verdict R0

**Définition** :
> Verdict = statut de viabilité structurelle d'un mécanisme Γ sur l'espace testé.

**Règle cardinale R0** :
> Une pathologie systématique suffit à invalider la viabilité.

### Verdicts autorisés

| Verdict | Signification exacte |
|---------|---------------------|
| **SURVIVES[R0]** | Non éliminé comme absurde, mécanisme potentiellement ajustable |
| **REJECTED[R0]** | Pathologie systématique détectée, mécanisme non viable en l'état |
| **WIP[R0-open]** | Indécidable (variance trop élevée, données insuffisantes) |

⚠️ **SURVIVES[R0] ≠ "intéressant"**

**Clarification terminologique** :

| Terme | Statut en R0 |
|-------|-------------|
| **SURVIVES[R0]** | Défini : non éliminé comme absurde |
| **Intéressant** | ❌ NON DÉFINI (jugement réservé à phases avec contexte) |
| **Prometteur** | ❌ Jugement réservé à R1+ |
| **Valide** | ❌ Jugement réservé à R2+ |

**SURVIVES[R0] signifie UNIQUEMENT** :
> Ce mécanisme aveugle Γ sur espace D testé ne déclenche pas de pathologie systématique suffisamment grave pour être éliminé sans examen approfondi.

---

## 12.9.2 Critères verdict (baseline)

### Critère 1 : Majorité configs saines

```python
majority_pct = (n_configs_healthy / n_total_configs) * 100

# Config healthy si :
test_score < threshold_test
```

**Interprétation** :
- Élevé : Γ fonctionne sur majorité configs testées
- Faible : Γ échoue systématiquement

---

### Critère 2 : Robustesse cross-D

```python
robustness_pct = (n_D_with_viable_config / n_total_D) * 100

# D viable si : ≥1 config healthy pour ce D
```

**Interprétation** :
- Élevé : Γ s'adapte à différents D
- Faible : Γ spécialisé ou fragile

---

### Critère 3 : Score global

```python
global_score = mean(all_test_scores)  # [0,1]
```

**Interprétation** :
- Faible : Pathologies concentrées (locales, ajustables)
- Élevé : Pathologies diffuses (systématiques)

## 12.9.4 Logique verdict : permissivité intentionnelle

### Règle SURVIVES (OU logique)

```python
if (global_score < survives.score_max OR
    robustness_pct >= survives.robustness_min * 100 OR
    majority_pct >= survives.majority_min * 100):
    verdict = "SURVIVES[R0]"
```

**Justification** :
La logique OU est **volontairement permissive** pour éviter l'élimination prématurée de mécanismes potentiellement ajustables. Le tri strict est reporté en R1.

**Philosophie R0** :
- R0 = filtre de **non-absurdité** (éliminer le clairement pathologique)
- R1 = tri **sélectif** (garder le prometteur avec contexte interprétatif)
- R2+ = validation **inter-domaines**

---

### Règle REJECTED (ET logique)

```python
elif (global_score > rejected.score_min AND
      robustness_pct < rejected.robustness_max * 100 AND
      majority_pct < rejected.majority_max * 100):
    verdict = "REJECTED[R0]"
```

**Condition** : Tous critères doivent échouer simultanément.

**Signification** : Pathologies systématiques sur tous axes (score, robustesse, majorité).

---

### Règle WIP (défaut)

```python
else:
    verdict = "WIP[R0-open]"
```

**Signification** : Ni clairement viable, ni clairement pathologique. Nécessite :
- Plus de données
- Configs alternatives
- Ou investigation manuelle

## 12.9.5 Algorithme verdict complet

```python
def compute_gamma_verdict(
    gamma_id: str,
    params_config_id: str,
    scoring_config_id: str,
    thresholds_config_id: str
) -> dict:
    """
    Calcule verdict global pour une gamma.
    
    Agrège TOUS test_scores sur TOUS runs.
    
    Returns:
        {
            'gamma_id': str,
            'params_config_id': str,
            'scoring_config_id': str,
            'thresholds_config_id': str,
            'majority_pct': float,
            'robustness_pct': float,
            'global_score': float,
            'verdict': str,
            'verdict_reason': str,
            'details': dict  # Décomposition complète
        }
    """
    # 1. Charger config thresholds
    thresholds = config_loader.load('thresholds', thresholds_config_id)
    threshold_test = thresholds['threshold_test']
    
    # 2. Récupérer tous test_scores pour cette gamma
    all_scores = db.query("""
        SELECT 
            ts.exec_id, 
            ts.test_name, 
            ts.test_score, 
            ts.test_weight,
            ts.pathology_flags,
            ts.critical_metrics,
            e.d_base_id, 
            e.modifier_id, 
            e.seed
        FROM TestScores ts
        JOIN Executions e ON ts.exec_id = e.id
        WHERE e.gamma_id = ?
          AND ts.params_config_id = ?
          AND ts.scoring_config_id = ?
    """, gamma_id, params_config_id, scoring_config_id)
    
    if not all_scores:
        return {
            'gamma_id': gamma_id,
            'params_config_id': params_config_id,
            'scoring_config_id': scoring_config_id,
            'thresholds_config_id': thresholds_config_id,
            'majority_pct': 0.0,
            'robustness_pct': 0.0,
            'global_score': 1.0,
            'verdict': 'WIP[R0-open]',
            'verdict_reason': 'Aucune donnée disponible',
            'details': {}
        }
    
    # 3. Calculer critère MAJORITÉ
    configs_healthy = [s for s in all_scores if s['test_score'] < threshold_test]
    majority_pct = (len(configs_healthy) / len(all_scores)) * 100
    
    # 4. Calculer critère ROBUSTESSE (grouper par D)
    d_bases = {}
    for score in all_scores:
        d_id = score['d_base_id']
        if d_id not in d_bases:
            d_bases[d_id] = []
        d_bases[d_id].append(score['test_score'])
    
    viable_d = sum(1 for scores in d_bases.values() 
                   if any(s < threshold_test for s in scores))
    robustness_pct = (viable_d / len(d_bases)) * 100 if d_bases else 0
    
    # 5. Calculer GLOBAL_SCORE
    global_score = np.mean([s['test_score'] for s in all_scores])
    
    # 6. Appliquer règles verdict
    survives = thresholds['survives']
    rejected = thresholds['rejected']
    
    # Logique OU pour SURVIVES
    survives_checks = {
        'score': global_score < survives['score_max'],
        'robustness': robustness_pct >= survives['robustness_min'] * 100,
        'majority': majority_pct >= survives['majority_min'] * 100
    }
    
    if any(survives_checks.values()):
        verdict = "SURVIVES[R0]"
        
        # Construire raison détaillée
        reasons = []
        if survives_checks['score']:
            reasons.append(f"global_score={global_score:.3f} < {survives['score_max']}")
        if survives_checks['robustness']:
            reasons.append(f"robustness={robustness_pct:.1f}% ≥ {survives['robustness_min']*100}%")
        if survives_checks['majority']:
            reasons.append(f"majority={majority_pct:.1f}% ≥ {survives['majority_min']*100}%")
        
        verdict_reason = "SURVIVES (logique OU): " + " OR ".join(reasons)
    
    # Logique ET pour REJECTED
    elif (global_score > rejected['score_min'] and
          robustness_pct < rejected['robustness_max'] * 100 and
          majority_pct < rejected['majority_max'] * 100):
        
        verdict = "REJECTED[R0]"
        verdict_reason = (
            f"REJECTED (logique ET): "
            f"global_score={global_score:.3f} > {rejected['score_min']} AND "
            f"robustness={robustness_pct:.1f}% < {rejected['robustness_max']*100}% AND "
            f"majority={majority_pct:.1f}% < {rejected['majority_max']*100}%"
        )
    
    # Sinon WIP
    else:
        verdict = "WIP[R0-open]"
        verdict_reason = (
            f"WIP (ni SURVIVES ni REJECTED): "
            f"global_score={global_score:.3f}, "
            f"robustness={robustness_pct:.1f}%, "
            f"majority={majority_pct:.1f}%"
        )
    
    # 7. Construire détails (pour rapport)
    # Identifier tests critiques
    critical_tests = [
        {
            'exec_id': s['exec_id'],
            'test_name': s['test_name'],
            'test_score': s['test_score'],
            'pathology_flags': json.loads(s['pathology_flags']) if s['pathology_flags'] else [],
            'critical_metrics': json.loads(s['critical_metrics']) if s['critical_metrics'] else []
        }
        for s in all_scores if s['test_score'] >= 0.8
    ]
    
    # Décomposition par D
    d_breakdown = {}
    for d_id, scores in d_bases.items():
        d_breakdown[d_id] = {
            'n_configs': len(scores),
            'n_healthy': sum(1 for s in scores if s < threshold_test),
            'mean_score': np.mean(scores),
            'min_score': np.min(scores),
            'max_score': np.max(scores)
        }
    
    details = {
        'n_total_configs': len(all_scores),
        'n_healthy_configs': len(configs_healthy),
        'n_total_d': len(d_bases),
        'n_viable_d': viable_d,
        'critical_tests': critical_tests,
        'd_breakdown': d_breakdown,
        'thresholds_used': {
            'threshold_test': threshold_test,
            'survives': survives,
            'rejected': rejected
        }
    }
    
    return {
        'gamma_id': gamma_id,
        'params_config_id': params_config_id,
        'scoring_config_id': scoring_config_id,
        'thresholds_config_id': thresholds_config_id,
        'majority_pct': majority_pct,
        'robustness_pct': robustness_pct,
        'global_score': global_score,
        'verdict': verdict,
        'verdict_reason': verdict_reason,
        'details': details
    }
```

## 12.9.6 Traçabilité verdict : rapport détaillé obligatoire

**Obligation normative** :
Tout verdict doit générer un rapport lisible explicitant :

1. **Valeurs critères** (majority, robustness, global_score)
2. **Seuils appliqués** (config thresholds)
3. **Tests critiques** (pathology_flags par test)
4. **Raison précise** (quel critère a déclenché verdict)
5. **Décomposition par D** (pour investiguer patterns)

## 12.9.7 Stockage db_results

```sql
CREATE TABLE GammaVerdicts (
  id INTEGER PRIMARY KEY,
  gamma_id TEXT NOT NULL,
  
  -- Configs utilisées (traçabilité complète)
  params_config_id TEXT NOT NULL,
  scoring_config_id TEXT NOT NULL,
  thresholds_config_id TEXT NOT NULL,
  
  -- Critères
  majority_pct REAL NOT NULL,
  robustness_pct REAL NOT NULL,
  global_score REAL NOT NULL,  -- [0,1] pathology score moyen
  
  -- Verdict
  verdict TEXT NOT NULL,  -- SURVIVES[R0] | WIP[R0-open] | REJECTED[R0]
  verdict_reason TEXT NOT NULL,
  
  -- Détails (JSON complet pour traçabilité)
  details TEXT NOT NULL,  -- JSON {n_configs, critical_tests, d_breakdown, ...}
  
  -- Metadata
  computed_at TEXT NOT NULL,
  
  UNIQUE(gamma_id, params_config_id, scoring_config_id, thresholds_config_id),
  FOREIGN KEY(params_config_id) REFERENCES ConfigRegistry(config_id),
  FOREIGN KEY(scoring_config_id) REFERENCES ConfigRegistry(config_id),
  FOREIGN KEY(thresholds_config_id) REFERENCES ConfigRegistry(config_id)
);

-- Index
CREATE INDEX idx_gamma_verdicts_lookup 
ON GammaVerdicts(gamma_id, params_config_id, scoring_config_id, thresholds_config_id);

CREATE INDEX idx_gamma_verdicts_gamma 
ON GammaVerdicts(gamma_id);

CREATE INDEX idx_gamma_verdicts_verdict 
ON GammaVerdicts(verdict);
```

## SECTION 15 : Glossaire

| Terme | Définition | Exemple |
|-------|------------|---------|
| **Registre** | Module contenant fonctions réutilisables | `algebra_registry.py` |
| **registry_key** | Identifiant unique fonction | `"algebra.matrix_norm"` |
| **run_metadata** | Métadonnées exécution | `{gamma_id, d_base_id, ...}` |
| **history** | Liste snapshots (~200) | `[state_0, ..., state_199]` |
| **snapshot** | UN état à une itération | `np.ndarray (10, 10)` |
| **post_processor** | Transformation post-calcul | `"round_4"` |
| **COMPUTATION_SPECS** | Spécifications métriques | `{'metric': {'registry_key': ...}}` |
| **APPLICABILITY_SPEC** | Contraintes techniques | `{'requires_rank': 2}` |

R : Voir Section 11.1 "Créer Nouveau Registre"

R : Non. Si besoin > 5, créer 2 tests séparés.

R : Voir Section 8.3 "Config YAML Override"

**Q5 : Que faire si validator manquant ?**  
R : Ajouter dans `VALIDATORS` (Section 12.1)

**Q6 : Comment déboguer "registry_key not found" ?**  
R : Vérifier chargement : `RegistryManager().list_available_functions()`

**Q7 : state_shape toujours disponible ?**  
R : Oui, déduit du premier snapshot avant applicabilité.
