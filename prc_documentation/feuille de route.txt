**EXHAUSTIVITÉ TOTALE AU RANG 0**

Chaque hypothèse Γ sera testée sur l'espace complet:
- **Toutes** les bases D applicables
- **Tous** les modificateurs standards
- **Toutes** les valeurs de paramètres pertinentes

## SECTION 1 : STRUCTURE DE DONNÉES MULTIDIMENSIONNELLE

### 1.1 Espace d'exploration

L'exploration R0 génère un hypercube de données à 6 dimensions:

```
DIMENSIONS = {
  Γ_id         : {GAM-001, GAM-002, ..., GAM-014}      (14 valeurs)
  Γ_params     : dict{param_name: value}               (variable par Γ)
  D_base_id    : {SYM-001, ..., SYM-006,               (13 valeurs)
                  ASY-001, ..., ASY-004,
                  R3-001, ..., R3-003}
  Modifier_id  : {M0, M1, M2, M3}                      (4 valeurs)
  Seed         : {1, 2, 3, 4, 5}                       (5 valeurs)
  Test_id      : {UNIV-001, ..., LOC-002}              (13+ tests)
}

RÉSULTAT par cellule = {
  status: "PASS" | "FAIL" | "NEUTRAL" | "ERROR",
  value: float,
  metadata: dict
}
```

### 1.2 Principe de non-mélange des cellules

**RÈGLE CRITIQUE** : Chaque cellule de l'hypercube est une exécution indépendante et atomique.

**Interdictions formelles** :
- ❌ Ne jamais moyenner des résultats de Γ_params différents
- ❌ Ne jamais agréger des D_base incompatibles (SYM vs ASY)
- ❌ Ne jamais comparer directement des tests de natures différentes

**Agrégations autorisées** :
- ✅ Moyenne sur seeds (même Γ_params, même D_base, même modifier)
- ✅ Score par test_family (UNIV, SYM, STR, etc.) sur même configuration
- ✅ Robustesse = % PASS sur D_base variés (même Γ_params, même modifier)

**Rationale** : Les dimensions ont des **sémantiques différentes**. Mélanger aveuglément détruit l'information structurelle. Chaque agrégation doit être **intentionnelle et documentée**.

**Exemple de mélange invalide** :
```
❌ score_global = mean([
    result(Γ-001, β=1.0, D-SYM-001, M0, seed=1, TEST-UNIV-001),
    result(Γ-001, β=2.0, D-ASY-002, M2, seed=3, TEST-SYM-001)
])
```
→ Mélange paramètres, bases incompatibles, tests différents.

**Exemple d'agrégation valide** :
```
✅ robustesse_β1 = mean_over_seeds([
    result(Γ-001, β=1.0, D-SYM-001, M0, seed=s, TEST-UNIV-001)
    for s in [1..5]
])
✅ score_D_SYM = mean_over_applicable_tests([
    robustesse(Γ-001, β=1.0, D-SYM-001, M0, test=T)
    for T in [UNIV-*, SYM-*, STR-*] if applicable
])
```

### 1.3 Schéma SQL formel

```sql
-- Table principale : exécutions atomiques
CREATE TABLE Executions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  gamma_id TEXT NOT NULL,              -- "GAM-001"
  gamma_params TEXT NOT NULL,          -- JSON: {"beta": 2.0}
  d_base_id TEXT NOT NULL,             -- "SYM-001"
  modifier_id TEXT NOT NULL,           -- "M1"
  seed INTEGER NOT NULL,
  max_iterations INTEGER NOT NULL,
  timestamp TEXT NOT NULL,
  execution_time_seconds REAL,
  status TEXT NOT NULL,                -- "COMPLETED" | "ERROR" | "TIMEOUT"
  UNIQUE(gamma_id, gamma_params, d_base_id, modifier_id, seed)
);

-- Table résultats tests
CREATE TABLE TestResults (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  exec_id INTEGER NOT NULL,
  test_name TEXT NOT NULL,             -- "TEST-UNIV-001"
  status TEXT NOT NULL,                -- "PASS" | "FAIL" | "NEUTRAL"
  value REAL,                          -- Valeur numérique du test
  metadata TEXT,                       -- JSON: détails additionnels
  FOREIGN KEY (exec_id) REFERENCES Executions(id)
);

-- Table historiques (optionnel, si stockage)
CREATE TABLE Histories (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  exec_id INTEGER NOT NULL,
  iteration INTEGER NOT NULL,
  state_blob BLOB,                     -- np.ndarray compressé
  FOREIGN KEY (exec_id) REFERENCES Executions(id)
);

-- Index pour requêtes rapides
CREATE INDEX idx_exec_gamma ON Executions(gamma_id, gamma_params);
CREATE INDEX idx_exec_d ON Executions(d_base_id);
CREATE INDEX idx_test_name ON TestResults(test_name);
```

## SECTION 2 : CATALOGUE DES BASES D^(base)

### 2.1 Bases symétriques rang 2

```
D-SYM-001: Matrice identité
  Dimensions: N ∈ {30, 100}
  Forme: I_N (diag=1, hors-diag=0)
  Usage: Test stabilité minimale, point fixe trivial
  Propriétés: symétrique, définie positive, sparse

D-SYM-002: Matrice aléatoire symétrique uniforme
  Dimensions: N ∈ {30, 100}
  Forme: A = (B + B^T)/2, B_ij ~ U[-1,1]
  Usage: Test diversité maximale, générique
  Propriétés: symétrique, bornes [-1,1]

D-SYM-003: Matrice aléatoire symétrique gaussienne
  Dimensions: N ∈ {30, 100}
  Forme: A = (B + B^T)/2, B_ij ~ N(0, σ=0.3)
  Usage: Test continuité, distribution normale
  Propriétés: symétrique, non bornée a priori

D-SYM-004: Matrice de corrélation aléatoire (SPD)
  Dimensions: N ∈ {30}
  Forme: A = C·C^T normalisée, C_ij ~ N(0,1)
  Usage: Test positivité définie
  Propriétés: symétrique, définie positive, diag=1

D-SYM-005: Matrice bande symétrique
  Dimensions: N = 50
  Forme: A_ij ≠ 0 ssi |i-j| ≤ 3, valeurs ~ U[-0.5,0.5]
  Usage: Test localité structurelle
  Propriétés: symétrique, sparse, bande

D-SYM-006: Matrice hiérarchique par blocs
  Dimensions: N = 50 (10 blocs 5×5)
  Forme: Blocs denses intra, sparse inter
  Usage: Test préservation structure modulaire
  Propriétés: symétrique, structure blocs
```

### 2.2 Bases asymétriques rang 2

```
D-ASY-001: Matrice asymétrique aléatoire uniforme
  Dimensions: N ∈ {30, 100}
  Forme: A_ij ~ U[-1,1] indépendants
  Usage: Test asymétrie générique
  Propriétés: asymétrique, bornes [-1,1]

D-ASY-002: Matrice triangulaire inférieure
  Dimensions: N = 50
  Forme: A_ij = U[-1,1] si i > j, sinon 0
  Usage: Test orientation directionnelle
  Propriétés: asymétrique, sparse, triangulaire

D-ASY-003: Matrice antisymétrique
  Dimensions: N = 50
  Forme: A = -A^T, A_ij ~ U[-1,1] pour i>j
  Usage: Test conservation antisymétrie
  Propriétés: antisymétrique (cas spécial asymétrique)

D-ASY-004: Matrice avec gradient directionnel
  Dimensions: N = 50
  Forme: A_ij = 0.1·(i-j) + U[-0.2,0.2]
  Usage: Test brisure symétrie avec structure
  Propriétés: asymétrique, gradient linéaire
```

### 2.3 Bases rang 3

```
D-R3-001: Tenseur aléatoire uniforme
  Dimensions: N = 20 (20×20×20)
  Forme: T_ijk ~ U[-1,1]
  Usage: Test générique rang 3
  Propriétés: aucune symétrie

D-R3-002: Tenseur symétrique partiel (indices 2-3)
  Dimensions: N = 20
  Forme: T_ijk = T_ikj, valeurs ~ U[-1,1]
  Usage: Test symétries partielles
  Propriétés: symétrie sur 2 indices

D-R3-003: Tenseur avec couplages locaux
  Dimensions: N = 15
  Forme: T_ijk ≠ 0 ssi |i-j|+|j-k|+|k-i| ≤ 5

  Propriétés: sparse, localité géométrique
```

### 2.4 Modificateurs standardisés

Chaque D^(base) sera testé avec 4 configurations:

```
M0: Base seule (aucun modificateur)

  Paramètres: σ = 0.05

M2: Base + bruit uniforme

  Usage: Test robustesse bruit borné

## SECTION 3 : CATALOGUE DES TESTS MODULAIRES

### 3.1 Tests universels (tous D, tous Γ)

```
TEST-UNIV-001: Évolution norme Frobenius
  Mesure: ||D||_F à chaque itération
  Output: {trend: "increasing"|"decreasing"|"stable"|"oscillating",
           initial: float, final: float, max: float}
  Verdict: PASS si trend ∈ {stable, oscillating} ET max < 1000
           FAIL si trend = increasing ET max > 1000 (explosion)
           NEUTRAL sinon

TEST-UNIV-002: Évolution diversité (std)
  Mesure: σ(D_flat) à chaque itération
  Output: {trend, initial, final, ratio_final/initial}
  Verdict: PASS si ratio ∈ [0.3, 3.0] (diversité préservée)
           FAIL si ratio < 0.1 (homogénéisation) OU > 10 (explosion)
           NEUTRAL sinon

TEST-UNIV-003: Convergence vers point fixe
  Mesure: ||D_{t+1} - D_t|| à chaque itération
  Output: {converged: bool, distance_final: float, 
           iterations_to_convergence: int}
  Verdict: PASS si converged ET distance_final > 0.01 (non-trivial)
           FAIL si converged ET distance_final < 0.001 (trivial)
           NEUTRAL si non converged

TEST-UNIV-004: Sensibilité conditions initiales
  Mesure: Distance entre trajectoires (5 seeds)
  Output: {mean_distance: float, std_distance: float}
  Verdict: PASS si std_distance < 0.5·mean_distance (robuste)
           FAIL si std_distance > 2·mean_distance (chaotique)
           NEUTRAL sinon
```

### 3.2 Tests symétrie (rang 2 uniquement)

```
TEST-SYM-001: Préservation symétrie
  Applicable: Si D initial symétrique (D-SYM-*)
  Mesure: max_t ||D_t - D_t^T|| / ||D_t||
  Output: {max_asymmetry: float, preserved: bool}
  Verdict: PASS si max_asymmetry < 1e-4
           FAIL si max_asymmetry > 1e-2
           NEUTRAL sinon

TEST-SYM-002: Création symétrie
  Applicable: Si D initial asymétrique (D-ASY-*)
  Mesure: ||D_final - D_final^T|| / ||D_final||
  Output: {initial_asym: float, final_asym: float, created: bool}
  Verdict: PASS si final_asym < 1e-4 ET initial_asym > 1e-2 (création)
           NEUTRAL sinon (pas de création attendue)

TEST-SYM-003: Amplification/réduction asymétrie
  Applicable: Tous D rang 2
  Mesure: ratio ||D - D^T|| / ||D + D^T|| initial vs final
  Output: {ratio_initial: float, ratio_final: float, trend}
  Verdict: NEUTRAL (observation pure, pas de verdict)
```

### 3.3 Tests structure (rang 2)

```
TEST-STR-001: Préservation rang
  Applicable: Matrices rang 2
  Mesure: rank(D_t) via SVD (seuil 1e-6)
  Output: {rank_initial: int, rank_final: int, trend}
  Verdict: PASS si |rank_final - rank_initial| ≤ 2
           FAIL si rank_final < 0.5·rank_initial (effondrement)
           NEUTRAL sinon

TEST-STR-002: Évolution spectre
  Applicable: Matrices symétriques uniquement
  Mesure: Valeurs propres λ_i à chaque itération
  Output: {eig_initial: array, eig_final: array,
           spectral_drift: float}
  Verdict: PASS si spectral_drift < 1.0
           FAIL si any(|λ_i| > 100) (explosion spectrale)
           NEUTRAL sinon

TEST-STR-003: Corrélations temporelles
  Applicable: Tous D
  Mesure: corr(D_flat(t), D_flat(t+k)) pour k ∈ {1,5,10}
  Output: {corr_1: float, corr_5: float, corr_10: float}
  Verdict: NEUTRAL (observation mémoire)
```

### 3.4 Tests bornes/contraintes

```
TEST-BND-001: Respect bornes initiales
  Applicable: Si D_base spécifie bornes
  Mesure: min(D_t), max(D_t) à chaque itération
  Output: {initial_bounds: [min, max], final_bounds: [min, max],
           violated: bool}
  Verdict: PASS si final dans 1.1× initial_bounds
           FAIL si final hors 2× initial_bounds
           NEUTRAL sinon

TEST-BND-002: Préservation positivité
  Applicable: Si D initial défini positif (D-SYM-004)
  Mesure: min(eig(D_t)) à chaque itération
  Output: {min_eig_initial: float, min_eig_final: float,
           positive_definite_preserved: bool}
  Verdict: PASS si min_eig_final > 1e-6
           FAIL si min_eig_final < -1e-6
           NEUTRAL si ∈ [-1e-6, 1e-6]
```

### 3.5 Tests localité

```
TEST-LOC-001: Propagation information
  Applicable: Tous D
  Méthode: Perturber D[i0,j0] += 0.1, mesurer effet sur |i-i0|+|j-j0|>k
  Output: {propagation_radius: float, propagation_speed: float}
  Verdict: NEUTRAL (observation)
  Note: Test coûteux, exécuter sur sous-échantillon

TEST-LOC-002: Préservation sparsité
  Applicable: Si D initial sparse (M3 ou D-SYM-005)
  Mesure: % éléments |D_ij| < ε (ε=1e-3) initial vs final
  Output: {sparsity_initial: float, sparsity_final: float}
  Verdict: PASS si sparsity_final > 0.5·sparsity_initial
           FAIL si sparsity_final < 0.1·sparsity_initial (densification)
           NEUTRAL sinon
```

## SECTION 4 : HYPOTHÈSES Γ RANG 0 AVEC GRILLES DE PARAMÈTRES

### 4.1 Famille markovienne pure

```
HYP-GAM-001: Saturation pure pointwise (tanh)
  FORME: T_{n+1}[i,j] = tanh(β · T_n[i,j])
  PARAMÈTRES: β (force de saturation)
  GRILLE: β ∈ {0.5, 1.0, 2.0, 5.0}
  D_APPLICABLES: Tous (SYM, ASY, R3)
  TESTS_PRIORITAIRES: UNIV-*, SYM-* (selon D), BND-001
  ATTENDU: Convergence monotone vers sign(D_ij), possible trivialité

HYP-GAM-002: Diffusion pure (Laplacien discret)
  FORME: T_{n+1}[i,j] = T_n[i,j] + α·(somme_voisins - 4·T_n[i,j])
  PARAMÈTRES: α (coefficient diffusion)
  GRILLE: α ∈ {0.01, 0.05, 0.1}
  D_APPLICABLES: SYM, ASY (rang 2 uniquement)
  TESTS_PRIORITAIRES: UNIV-002 (diversité), LOC-001 (propagation)
  ATTENDU: Homogénéisation, perte diversité

HYP-GAM-003: Croissance exponentielle pointwise
  FORME: T_{n+1}[i,j] = T_n[i,j] · exp(γ)
  PARAMÈTRES: γ (taux de croissance)
  GRILLE: γ ∈ {0.01, 0.05, 0.1}
  D_APPLICABLES: Tous
  TESTS_PRIORITAIRES: UNIV-001 (norme), BND-001 (bornes)
  ATTENDU: Explosion, REJECTED[GLOBAL] probable

HYP-GAM-004: Décroissance exponentielle pointwise
  FORME: T_{n+1}[i,j] = T_n[i,j] · exp(-γ)
  PARAMÈTRES: γ (taux de décroissance)
  GRILLE: γ ∈ {0.01, 0.05, 0.1}
  D_APPLICABLES: Tous
  TESTS_PRIORITAIRES: UNIV-001, UNIV-002, UNIV-003
  ATTENDU: Convergence vers 0, trivialité

HYP-GAM-005: Oscillateur harmonique linéaire
  FORME: T_{n+1} = cos(ω)·T_n - sin(ω)·T_{n-1}
  PARAMÈTRES: ω (fréquence angulaire)
  GRILLE: ω ∈ {π/8, π/4, π/2}
  D_APPLICABLES: SYM, ASY
  TESTS_PRIORITAIRES: UNIV-003 (convergence), STR-003 (mémoire)
  ATTENDU: Oscillations périodiques, pas de complexité émergente
  NOTE: Non-markovien ordre 1
```

### 4.2 Famille non-markovienne (mémoire)

```
HYP-GAM-006: Saturation avec mémoire ordre-1
  FORME: T_{n+1} = tanh(β·T_n + α·(T_n - T_{n-1}))
  PARAMÈTRES: {β: force sat, α: poids mémoire}
  GRILLE: β ∈ {1.0, 2.0}, α ∈ {0.1, 0.3, 0.5}
           → 6 combinaisons
  D_APPLICABLES: Tous
  TESTS_PRIORITAIRES: UNIV-004 (sensibilité), STR-003 (mémoire)
  ATTENDU: Inertie temporelle, possibilité de non-trivialité

HYP-GAM-007: Régulation par moyenne glissante
  FORME: T_{n+1}[i,j] = (1-ε)·T_n[i,j] + ε·mean(voisins_8(T_n))
  PARAMÈTRES: ε (force de régulation)
  GRILLE: ε ∈ {0.05, 0.1, 0.2}
  D_APPLICABLES: SYM, ASY (rang 2)
  TESTS_PRIORITAIRES: UNIV-002 (lissage), LOC-001 (propagation)
  ATTENDU: Homogénéisation douce locale

HYP-GAM-008: Mémoire différentielle
  FORME: T_{n+1} = T_n + γ·(T_n - T_{n-1}) + saturation(β)
  PARAMÈTRES: {γ: inertie, β: saturation}
  GRILLE: γ ∈ {0.1, 0.3}, β ∈ {1.0, 2.0}
           → 4 combinaisons
  D_APPLICABLES: SYM, ASY
  TESTS_PRIORITAIRES: UNIV-001 (stabilité), UNIV-004
  ATTENDU: "Friction informationnelle", oscillations contrôlées
```

### 4.3 Famille stochastique

```
HYP-GAM-009: Saturation + bruit additif
  FORME: T_{n+1} = tanh(β·T_n) + σ·ε, ε ~ N(0,1)
  PARAMÈTRES: {β: saturation, σ: amplitude bruit}
  GRILLE: β ∈ {1.0, 2.0}, σ ∈ {0.01, 0.05}
           → 4 combinaisons
  D_APPLICABLES: Tous
  TESTS_PRIORITAIRES: UNIV-004 (sensibilité), UNIV-002 (équilibre)
  ATTENDU: Équilibre stochastique ou diffusion aléatoire

HYP-GAM-010: Bruit multiplicatif
  FORME: T_{n+1} = T_n · (1 + σ·ε) avec saturation
  PARAMÈTRES: σ (amplitude bruit multiplicatif)
  GRILLE: σ ∈ {0.01, 0.05, 0.1}
  D_APPLICABLES: Tous
  TESTS_PRIORITAIRES: BND-001, UNIV-001 (explosion)
  ATTENDU: Amplification sélective, risque d'avalanche

HYP-GAM-011: Processus de branchement tensoriel
  FORME: T_{n+1}[i,j] = Σ_k A[i,j,k]·T_n[k,j] + bruit
  PARAMÈTRES: {λ: taux branchement, σ: bruit}
  GRILLE: λ ∈ {0.9, 1.0, 1.1}, σ ∈ {0.01, 0.05}
           → 6 combinaisons
  D_APPLICABLES: R3 (tenseur rang 3 nécessaire)
  TESTS_PRIORITAIRES: UNIV-001, UNIV-002, STR-001
  ATTENDU: Criticalité possible, émergence patterns
  NOTE: Complexité élevée
```

### 4.4 Famille structurelle

```
HYP-GAM-012: Préservation symétrie forcée
  FORME: T_{n+1} = (F(T_n) + F(T_n)^T) / 2, F = saturation
  PARAMÈTRES: β (force saturation dans F)
  GRILLE: β ∈ {1.0, 2.0, 5.0}
  D_APPLICABLES: SYM, ASY
  TESTS_PRIORITAIRES: SYM-001, SYM-002 (création symétrie)
  ATTENDU: Réparation artificielle, robustesse au bruit asymétrique

HYP-GAM-013: Renforcement hebbien local
  FORME: T_{n+1}[i,j] = T_n[i,j] + η·Σ_k T_n[i,k]·T_n[k,j]
  PARAMÈTRES: η (taux d'apprentissage)
  GRILLE: η ∈ {0.001, 0.01, 0.05}
  D_APPLICABLES: ASY (asymétrique favorisé), R3
  TESTS_PRIORITAIRES: UNIV-001 (explosion), STR-001 (rang)
  ATTENDU: Émergence clusters, croissance non-linéaire

HYP-GAM-014: Projection sur sous-espace contraint
  FORME: T_{n+1} = P·saturation(T_n)·P^T, P projecteur
  PARAMÈTRES: {dim_subspace: int, β: saturation}
  GRILLE: dim ∈ {N/2, N/4}, β ∈ {1.0, 2.0}
           → 4 combinaisons
  D_APPLICABLES: SYM, ASY
  TESTS_PRIORITAIRES: STR-001 (rang), UNIV-002 (conservation info)
  ATTENDU: Réduction dimension, perte info hors-sous-espace

## SECTION 5 : ORGANISATION PAR TOY MODELS

### 5.1 Principe de groupement

Chaque hypothèse Γ génère **1 à 5 Toy Models** selon la complexité de sa grille de paramètres et l'espace D applicable.

**Règle de division** :
- Si grille_params × D_applicables ≤ 50 combinaisons → 1 TM
- Si 50 < combinaisons ≤ 150 → 2-3 TM (par famille D ou range de params)
- Si > 150 combinaisons → 4-5 TM (par sous-grille)

### 5.2 Exemples de découpage

**Cas simple : HYP-GAM-001**
```
TM-GAM-001: Saturation pure exhaustive
  PARAMÈTRES: β ∈ {0.5, 1.0, 2.0, 5.0}
  D_BASES: Tous (SYM, ASY, R3) = 13 bases
  MODIFIERS: M0, M1, M2, M3
  SEEDS: 1..5
  
  Total runs = 4 params × 13 D × 4 M × 5 seeds = 1040 exécutions
  
  Structure TM unique car < 150 combinaisons de (params × D)
```

**Cas complexe : HYP-GAM-006**
```
PARAMÈTRES: β ∈ {1.0, 2.0}, α ∈ {0.1, 0.3, 0.5} → 6 combinaisons
D_BASES: 13 bases → 78 combinaisons (params × D)

Division en 2 TM:

TM-GAM-006-A: Mémoire faible
  PARAMÈTRES: α ∈ {0.1, 0.3}, β ∈ {1.0, 2.0}
  D_BASES: Tous
  Total: 4 params × 13 D × 4 M × 5 seeds = 1040 runs

TM-GAM-006-B: Mémoire forte
  PARAMÈTRES: α = 0.5, β ∈ {1.0, 2.0}
  D_BASES: Tous
  Total: 2 params × 13 D × 4 M × 5 seeds = 520 runs
```

## SECTION 7 : CRITÈRES D'ÉVALUATION MULTIPLES

### 7.1 Critère de Majorité

**Définition** : Pour un Γ donné avec grille de paramètres, combien de configurations (params × D × M) passent les tests ?

```
Majorité_Γ = (Nombre de configs avec verdict PASS) / (Nombre total de configs)

Classification:
  - Majorité ≥ 70% → Score_majorité = 4/4
  - Majorité 50-70% → Score_majorité = 3/4
  - Majorité 30-50% → Score_majorité = 2/4
  - Majorité < 30% → Score_majorité = 1/4
```

**Usage** : Identifier Γ qui "marchent souvent" vs "marchent rarement".

### 7.2 Critère de Robustesse

**Définition** : Pour un Γ donné, sur combien de D_base différents obtient-on au moins une config viable ?

```
Robustesse_Γ = (Nombre de D_base avec ≥1 config PASS) / (Nombre total D_base testés)

Classification:
  - Robustesse ≥ 80% → Score_robustesse = 4/4
  - Robustesse 60-80% → Score_robustesse = 3/4
  - Robustesse 40-60% → Score_robustesse = 2/4
  - Robustesse < 40% → Score_robustesse = 1/4
```

**Usage** : Identifier Γ "universels" vs "spécialisés à certains D".

### 7.3 Critère de Score Agrégé

**Définition** : Pour chaque exécution, calculer score normalisé basé sur tous les tests.

```
Score_exec = Σ_test (weight_test × normalized_value_test) / Σ weights

OÙ:
  weight_test défini par importance:
    - UNIV-001, UNIV-002, UNIV-003: weight = 2.0 (critiques)
    - SYM-*, STR-*: weight = 1.5 (structurels)
    - BND-*, LOC-*: weight = 1.0 (secondaires)
  
  normalized_value_test:
    - PASS: 1.0
    - NEUTRAL: 0.5
    - FAIL: 0.0

Score_Γ = median(Score_exec sur toutes configs)

Classification:
  - Score ≥ 15/20 → Excellent
  - Score 12-15/20 → Bon (SURVIVES[R0])
  - Score 8-12/20 → Moyen (WIP[R0-closed])
  - Score < 8/20 → Mauvais (REJECTED[R0])
```

**Usage** : Classement global prenant en compte qualité pondérée.

### 7.4 Synthèse décisionnelle

**Verdict final pour chaque Γ** combinant les 3 critères:

```
Si Score ≥ 12/20 ET Robustesse ≥ 60% ET Majorité ≥ 50%:
  → SURVIVES[R0]

Si Score < 8/20 OU Robustesse < 40% OU Majorité < 20%:
  → REJECTED[R0]

Si conditions intermédiaires:
  → WIP[R0-closed]

Si échecs catastrophiques (explosion, NaN) sur >80% configs:
  → REJECTED[GLOBAL]
```

**Tous ces critères seront calculés** et documentés dans les rapports, permettant analyses multidimensionnelles post-hoc.

### 10.2 Script batch_runner principal

```python
# prc_automation/batch_runner.py

"""
Orchestration exhaustive exécutions R0.

Usage:
    python batch_runner.py --phase 0 --tm GAM-001
    python batch_runner.py --phase 1 --all
    python batch_runner.py --phase 2 --gamma-list GAM-006,GAM-009
"""

import argparse
from core.kernel import run_kernel
from core.state_preparation import prepare_state
from database import insert_execution, insert_test_results
from tests.utilities import run_all_applicable_tests

def execute_single_run(gamma_id, gamma_params, d_base_id, 
                       modifier_id, seed, db_conn):
    """Exécute 1 cellule de l'hypercube."""
    
    # Préparer D
    D_base = load_d_base(d_base_id)
    modifier = load_modifier(modifier_id, seed)
    D = prepare_state(D_base, [modifier] if modifier else [])
    
    # Charger Γ
    gamma = load_gamma(gamma_id, gamma_params)
    
    # Exécuter kernel
    history = []
    try:
        for i, state in run_kernel(D, gamma, max_iterations=2000, 
                                    record_history=True):
            if i % 10 == 0:
                history.append(state.copy())
        
        status = "COMPLETED"
    except Exception as e:
        status = "ERROR"
        log_error(e)
    
    # Enregistrer exécution
    exec_id = insert_execution(db_conn, gamma_id, gamma_params,
                               d_base_id, modifier_id, seed, status)
    
    # Appliquer tous les tests applicables
    if status == "COMPLETED":
        test_results = run_all_applicable_tests(
            history, D, d_base_id, gamma_id
        )
        insert_test_results(db_conn, exec_id, test_results)
    
    return exec_id, status

def run_tm_exhaustive(tm_id, db_conn):
    """Exécute tous les runs d'un TM."""
    
    # Charger spécification TM
    tm_spec = load_tm_specification(tm_id)
    
    total_runs = (len(tm_spec.gamma_params_grid) × 
                  len(tm_spec.d_bases) × 
                  len(tm_spec.modifiers) × 
                  len(tm_spec.seeds))
    
    print(f"TM {tm_id}: {total_runs} runs à exécuter")
    
    completed = 0
    for gamma_params in tm_spec.gamma_params_grid:
        for d_base_id in tm_spec.d_bases:
            for modifier_id in tm_spec.modifiers:
                for seed in tm_spec.seeds:
                    
                    # Vérifier si déjà exécuté
                    if exists_in_db(db_conn, tm_spec.gamma_id, 
                                   gamma_params, d_base_id, 
                                   modifier_id, seed):
                        print(f"Skip (déjà exécuté)")
                        completed += 1
                        continue
                    
                    # Exécuter
                    exec_id, status = execute_single_run(
                        tm_spec.gamma_id, gamma_params,
                        d_base_id, modifier_id, seed, db_conn
                    )
                    
                    completed += 1
                    print(f"Progress: {completed}/{total_runs} "
                          f"({100*completed/total_runs:.1f}%)")
    
    print(f"TM {tm_id} terminé: {completed} runs")

# Main avec gestion phases
if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument("--phase", type=int, choices=[0,1,2])
    parser.add_argument("--tm", type=str, help="TM-GAM-XXX")
    parser.add_argument("--all", action="store_true")
    args = parser.parse_args()
    
    db = connect_database()
    
    if args.phase == 0:
        # Phase calibration: 1 TM pilote
        run_tm_exhaustive("TM-GAM-001", db)
    
    elif args.phase == 1:
        # Phase balayage: tous TM avec params nominaux
        for tm_id in get_all_tm_phase1():
            run_tm_exhaustive(tm_id, db)
    
    elif args.phase == 2:
        # Phase exploration: TM prometteurs grille complète
        promising_tms = get_promising_tms(db, score_threshold=10)
        for tm_id in promising_tms:
            run_tm_exhaustive(tm_id, db)
```

- Milestone (ex: 25%, 50%, 75% complété)
