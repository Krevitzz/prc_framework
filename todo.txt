## PHASE 7 — MÉTA-ANALYSE TESTS (fin r0 - DIFFÉRÉ)

### Objectif
Transformer axe test en méta-analyse des autres axes.

### Vision
Tests ne se profilent plus sous eux-mêmes, mais mesurent leur sensibilité relative aux différents axes causaux.

### Exemple structure cible
```python
test_profiles = {
    'SPE-001': {
        'gamma_sensitivity': 0.85,      # η² gamma
        'encoding_sensitivity': 0.12,   # η² encoding
        'modifier_sensitivity': 0.03,   # η² modifier
        'yaml_sensitivity': 0.00,       # η² yaml
        'best_discriminated_axis': 'gamma',
        'interpretation': 'Test dominé par choix gamma'
    },
    'SYM-001': {
        'gamma_sensitivity': 0.42,
        'encoding_sensitivity': 0.78,   # Sensible SYM vs ASY
        'modifier_sensitivity': 0.05,
        'best_discriminated_axis': 'encoding'
    }
}
```

### Utilité fin r0
- Identifier tests pertinents par axe analytique
- Détecter redondances tests
- Optimiser batteries futures

### Statut
**DIFFÉRÉ fin r0** : Nécessite base massive R0 complète pour calculs sensibilités robustes.

### Actions R0
- Désactiver profiling tests actuel (discriminant_power absurde)
- Conserver infrastructure pour R1

#### 10.2.1 Refonte Batch Runner-quick-checking

**Execution Registry (quick check léger)**
```json
// prc_database/execution_registry_r0.json
{
    "loaded_files": [
        "gamma_hyp_001.py",
        "gamma_hyp_002.py",
        "rank2_symmetric.py",
        "noise.py",
        "test_uni_001.py",
        "params_baseline.yaml"
    ],
    "last_update": "2026-01-22T15:30:00"
}
```

**Logique skip** :
- Découverte fichiers actifs courants
- Si fichier absent registry → append DB + registry
- Si tous fichiers présents → skip --brut
- Registry = quick check, PAS validation intégrité DB

**Scope registry** :
- ~100 lignes max (fichiers sources uniques)
- Évite 52k queries DB pour détection différentielle
- Debugging intégrité DB = hors scope (manuel si besoin)

#### 10.2.2 Refonte Batch Runner
- Une seule commande (plus de --mode)
- Découverte automatique fichiers actifs
- Détection combinaisons manquantes (append intelligent)
- Logique différentielle :
  - Nouveau test → append db_results_r*, rapports
  - Nouveau modifier/encoding/gamma → append db_raw_r*, db_results_r*, rapports
- Tags fichiers [R0], [R1], etc. pour détermination phase

#### 10.3 Migration Discovery
- Fusion discovery.py + discovery_entities.py → data_loading.py

#### 10.4 Utilitaires Bases Données
