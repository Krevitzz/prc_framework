## SECTION 1-3 : Fondations

### Axiomes (A1-A3)
- A1: Dissymétrie informationnelle D irréductible
- A2: Mécanisme Γ agissant sur D
- A3: Aucun Γ stable ne peut annuler D complètement

### Hiérarchie niveaux
```
L0=ontological → L1=epistemic → L2=theoretical → L3=operative → L4=documentary
Règle : L(n) ne référence que L(≤n-1)
```

### Core = 2 fonctions aveugles
```python
prepare_state(base, modifiers) → np.ndarray
run_kernel(state, gamma, ...) → Generator[iteration, state]
```

**Règles K1-K5** : Core aveugle (pas de validation, pas de sémantique, pas de classes)

## SECTION 11-12 : Pipeline opérationnel

### Workflow
```
1. Formulation HYP → 2. Spécification D/Γ → 3. Création TM
→ 4. Exécution kernel (db_raw) → 5. Tests+scoring (db_results)
→ 6. Discussion → 7. Mise à jour STATUS → 8. CON si pertinent
→ 9. Nouvelle HYP[R1] si pertinent
```

### Architecture batch_runner

**Principe** : 1 point d'entrée, 3 modes, chaînage auto

```bash
# Mode 1 : Collecte données
batch_runner --brut --gamma GAM-001
# → Exécute kernel, stocke db_raw (append-only, immuable)

# Mode 2 : Application tests
batch_runner --test --gamma GAM-001 --params default --scoring default
# → Vérifie db_raw, auto-lance --brut si manquant
# → Applique tests, calcule scores, stocke db_results

# Mode 3 : Génération verdicts
batch_runner --verdict --gamma GAM-001 --params default --scoring default --thresholds strict
# → Vérifie db_results, auto-lance --test si manquant
# → Calcule 3 critères, applique seuils, stocke verdict

# Raccourci
batch_runner --all --gamma GAM-001 --params default --scoring default --thresholds strict
```

**Règle non-reruns** : Toute modification scoring/verdict SANS réexécuter kernel

### Bases de données

**db_raw** (immuable, append-only) :
```sql
Executions   -- Métadonnées runs
Snapshots    -- États sauvegardés (compressed)
Metrics      -- Métriques brutes par iteration
```

**db_results** (rejouable, versionné) :
```sql
TestObservations  -- Résultats bruts tests
TestScores        -- Scores calculés (config_id)
GammaVerdicts     -- Verdicts agrégés (config_id + threshold_id)
```

## SECTION 12.4 : Tests - Format dict obligatoire

**Principe** : Tests OBSERVENT (pas de jugement), scoring INTERPRÈTE

### Status
- `SUCCESS` : Test exécuté normalement
- `ERROR` : Erreur technique → ARRÊT BATCH (bug code)
- `NOT_APPLICABLE` : Test invalide pour contexte (pas pénalisé)
- `SKIPPED` : Métrique spécifique omise (reste du test OK)

### Transitions factuelles (exemples)
```
"stable"      # Dans ±10% initial
"increasing"  # Croissance monotone
"explosive"   # Croissance > seuil
"converged"   # Distance successive < ε
"preserved"   # Propriété maintenue
"destroyed"   # Propriété perdue
```

### Interdictions formelles
```python
# ❌ JAMAIS dans retour test
'passed': bool
'failed': bool
'verdict': str
'score': float
'weight': float

# ❌ JAMAIS en code (YAML obligatoire)
if norm > 1000.0:  # Hardcodé
    ...

# ✅ TOUJOURS
params = load_test_parameters('UNIV-001')
if norm > params['explosion_threshold']:
    ...
```

### Architecture configs
```

  thresholds_strict_v1.yaml

config/tests/UNIV-001/      # Overrides spécifiques
```

### Verdict : 3 critères → 1 décision

**Critères** :
```python
majority_pct = (n_configs_pass / n_total_configs) * 100
robustness_pct = (n_D_with_viable_config / n_total_D) * 100
global_score = mean(all_config_scores)  # 0-1
```

**Logique (baseline)** :
```yaml
# SURVIVES[R0] : OU logique (1 critère suffit)
survives:
  score_min: 12.0      # /20
  robustness_min: 60.0 # %
  majority_min: 50.0   # %

# FLAGGED_FOR_REVIEW : ET logique (tous échouent)
flagged:
  score_max: 8.0
  robustness_max: 40.0
  majority_max: 20.0

# WIP[R0-closed] : Ni SURVIVES ni FLAGGED
```

**Verdicts** :
- `SURVIVES[R0]` : Non éliminé, viable pour exploration/composition
- `WIP[R0-closed]` : Ambigu, mise en attente (pas de promesse R1)
- `FLAGGED_FOR_REVIEW` : Échec probable, review manuelle obligatoire avant REJECTED

## SECTION 0 : Principes Directeurs

### 0.1. Philosophie

Le système à registres transforme le pipeline en un framework **extensible par conception**. Chaque registre est un **module autonome** encapsulant un domaine mathématique.

### 0.2. Règles Fondamentales

**R0-1** : Extensibilité sans modification du core  
**R0-2** : Découverte dynamique des registres au démarrage  
**R0-3** : Validation statique des signatures avant exécution  
**R0-4** : Documentation intégrée auto-générée  
**R0-5** : Aucun mode legacy - architecture unique

### 0.3. Terminologie Normalisée

| Terme | Définition |
|-------|------------|
| **run_metadata** | Métadonnées d'une exécution : {gamma_id, d_base_id, modifier_id, seed, state_shape} |
| **history** | Liste complète des snapshots d'un run (~200 états) |
| **snapshot** | UN état à une itération donnée (np.ndarray) |
| **registry** | Module contenant des fonctions mathématiques réutilisables |
| **registry_key** | Identifiant unique "registre.fonction" (ex: "algebra.matrix_norm") |
| **post_processor** | Transformation appliquée au résultat brut d'une métrique |

## SECTION 1 : Structure des Tests

### 1.1. Format Obligatoire
# prc_documentation/templates/test_xxx_yyy.py


### 1.2. Règles Strictes

**R1-1** : TEST_ID format "CAT-NNN" (3 lettres, tiret, 3 chiffres)  
**R1-2** : TEST_VERSION = "5.4" obligatoire  
**R1-3** : Docstring complète obligatoire (objectif + métriques + exclusions)  
**R1-4** : COMPUTATION_SPECS contient 1 à 5 métriques (pas 0, pas 10)  
**R1-5** : Chaque métrique a 'registry_key' et 'default_params' obligatoires  
**R1-6** : 'post_process' doit être une clé de POST_PROCESSORS (pas lambda string)  
**R1-7** : Aucun attribut FORMULAS (obsolète)

### 1.3. Validation

```python
# Validation automatique par discovery.py
def validate_test_structure(test_module):
    assert hasattr(test_module, 'TEST_ID')
    assert hasattr(test_module, 'TEST_VERSION')
    assert test_module.TEST_VERSION == "5.4"
    assert hasattr(test_module, 'APPLICABILITY_SPEC')
    assert hasattr(test_module, 'COMPUTATION_SPECS')
    assert not hasattr(test_module, 'FORMULAS')  # Obsolète
    assert 1 <= len(test_module.COMPUTATION_SPECS) <= 5
    
    for metric_name, spec in test_module.COMPUTATION_SPECS.items():
        assert 'registry_key' in spec
        assert 'default_params' in spec
        assert '.' in spec['registry_key']  # Format registre.fonction
```

## SECTION 7 : Workflow Complet

### 7.1. Pipeline

```
batch_runner --test --gamma GAM-001 --params default_v1
    ↓
[1] discovery.scan_tests()
    → {test_id: module} (COMPUTATION_SPECS obligatoire)
    ↓
[2] db.query("SELECT gamma_id, d_base_id, modifier_id, seed FROM Executions WHERE gamma_id=?")
    → Liste runs
    ↓
[3] Pour chaque run:
        run_metadata = {gamma_id, d_base_id, modifier_id, seed}
        ↓
        first_snapshot = db.load_first_snapshot(run_metadata)
        run_metadata['state_shape'] = first_snapshot.shape
        ↓
        Pour chaque test:
            ↓
            [4] applicability.check(test, run_metadata)
                ├─ Non → store NOT_APPLICABLE
                └─ Oui → continue
                    ↓
                    [5] history = db.load_all_snapshots(run_metadata)
                    ↓
                    [6] result = test_engine.execute_test(test, run_metadata, history, params_config_id)
                        ↓
                        RegistryManager.validate_computation_spec()
                        ↓
                        Pour chaque snapshot:
                            Pour chaque métrique:
                                function = registry.get_function(registry_key)
                                value = function(snapshot, **params)
                                post_process = POST_PROCESSORS[key]
                                final_value = post_process(value)
                                buffer.append(final_value)
                        ↓
                        Calculer statistics/evolution
                        ↓
                        return result
                    ↓
                    [7] store_result(result) → db_results
```

### 7.2. Optimisations

**O1** : Cache RegistryManager (fonctions)  
**O2** : Cache TestEngine (computations préparées)  
**O3** : Chargement state_shape = 1 snapshot uniquement  
**O4** : Chargement history = tous snapshots seulement si applicable

## SECTION 9 : Anti-Patterns

### 9.1. Tests

**AP1** : Trop de métriques
```python
# ❌ MAUVAIS (10 métriques)
COMPUTATION_SPECS = {
    'metric1': ...,
    'metric2': ...,
    # ... 10 au total
}

# ✅ BON (3 métriques bien choisies)
COMPUTATION_SPECS = {
    'metric1': ...,  # Principal
    'metric2': ...,  # Complémentaire
    'metric3': ...,  # Validation
}
```

**AP2** : Documentation absente
```python
# ❌ MAUVAIS
"""Test symétrie."""

# ✅ BON
"""
Évolution asymétrie matrices.

Objectif: ...
Métriques: ...
Exclusions: ...
"""
```

**AP3** : FORMULAS legacy
```python
# ❌ OBSOLÈTE
FORMULAS = {
    'metric': lambda state: np.linalg.norm(state),
}

# ✅ CORRECT
COMPUTATION_SPECS = {
    'metric': {
        'registry_key': 'algebra.matrix_norm',
        'default_params': {'norm_type': 'frobenius'},
    }
}
```

### 9.2. Registres

**AP4** : Fonction impure
```python
# ❌ MAUVAIS (modifie state)
def bad_function(state):
    state[0, 0] = 0
    return np.sum(state)

# ✅ BON (pure)
def good_function(state):
    return float(np.sum(state**2))
```

**AP5** : Docstring absente
```python
# ❌ MAUVAIS
def func(state, p):
    return state[0, 0]

# ✅ BON
def func(state, parameter=1.0):
    """
    Description complète.
    
    Args:
        state: Tenseur
        parameter: Description
    
    Returns:
        float: Valeur calculée
    """
    return float(state[0, 0] * parameter)
```

# SECTION 12.8 : SCORING - DÉTECTION PATHOLOGIES R0

## 12.8.1 Principe fondamental

En phase R0, **le scoring ne mesure pas des qualités, il détecte des pathologies structurelles**.

### Convention normalisée

```python
pathology_score ∈ [0, 1]
# 0 = aucun signal pathologique détecté (sain)
# 1 = pathologie maximale détectée (toxique)
```

**Affichage** : Les rapports peuvent afficher sur `/20` pour lisibilité (score × 20), mais **tous calculs internes utilisent [0,1]**.

### Interdictions formelles

- ❌ Interpréter un score élevé comme "bon"
- ❌ Optimiser pour maximiser un score
- ❌ Attribuer un "sens" positif aux métriques
- ❌ Compenser des pathologies critiques par moyenne

### Rôle des scores

**Les scores sont des observables tangibles**, pas des jugements. Ils permettent :
1. **Quantifier** une déviation par rapport à un seuil
2. **Comparer** configurations
3. **Détecter patterns** cross-runs (rôle verdict)

---

## 12.8.2 Typologie des pathologies (S1-S4 + MAPPING)

### TYPE S1 : COLLAPSE (seuil bas)

**Définition** : Valeur trop faible indique effondrement structural.

**Métriques typiques** :
- `entropy` : Perte diversité informationnelle
- `variance` : Collapse vers valeur unique
- `rank_estimate` : Perte dimensionnalité
- `spectral_gap` : Perte séparation modale

**Formule** :
```python
if mode == 'hard':
    score = 1.0 if value < threshold_low else 0.0
else:  # soft (default)
    if value >= threshold_low:
        score = 0.0
    elif value <= critical_low:
        score = 1.0
    else:
        score = (threshold_low - value) / (threshold_low - critical_low)

flag = value < critical_low
```

**Configuration YAML** :
```yaml
entropy:
  pathology_type: "S1_COLLAPSE"
  threshold_low: 0.1
  critical_low: 0.01
  weight: 2.0
  mode: "soft"
```

---

### TYPE S2 : EXPLOSION (seuil haut)

**Définition** : Valeur trop élevée indique divergence non contrôlée.

**Métriques typiques** :
- `spectral_radius` : Divergence itérative
- `condition_number` : Instabilité numérique
- `frobenius_norm` : Croissance explosive
- `gradient_magnitude` : Variations brutales

**Formule** :
```python
if mode == 'hard':
    score = 1.0 if value > threshold_high else 0.0
else:  # soft (default)
    if value <= threshold_high:
        score = 0.0
    elif value >= critical_high:
        score = 1.0
    else:
        score = (value - threshold_high) / (critical_high - threshold_high)

flag = value > critical_high
```

**Configuration YAML** :
```yaml
frobenius_norm:
  pathology_type: "S2_EXPLOSION"
  threshold_high: 1000.0
  critical_high: 10000.0
  weight: 2.0
  mode: "soft"
```

---

### TYPE S3 : PLATEAU (intervalle toxique)

**Définition** : Valeur trop stable ou uniformisée, perte structure exploitable.

**Métriques typiques** :
- `uniformity ≈ 1` : Distribution trop plate
- `sparsity ≈ 0` : Aucune parcimonie
- `symmetry_score ≈ 1` : Symétrie non désirée

**Formule** :
```python
interval = [interval_low, interval_high]

if interval[0] <= value <= interval[1]:
    score = 1.0
    flag = True
else:
    dist = min(abs(value - interval[0]), abs(value - interval[1]))
    interval_width = interval[1] - interval[0]
    score = max(0, 1 - dist / max(interval_width, 0.1))
    flag = False
```

**Configuration YAML** :
```yaml
uniformity:
  pathology_type: "S3_PLATEAU"
  interval_toxic: [0.9, 1.0]
  weight: 1.5
  mode: "soft"
```

---

### TYPE S4 : INSTABILITY (variation différentielle)

**Définition** : Changements trop rapides sous perturbation minime.

**Métriques typiques** :
- `Δentropy` : Volatilité entropique
- `Δrank` : Instabilité rang
- `volatility` : Variance des variations

**Formule** :
```python
delta = abs(value_t - value_t_prev)
score = max(0, min(1, delta / delta_critical))
flag = delta > delta_critical
```

**Configuration YAML** :
```yaml
evolution_volatility:
  pathology_type: "S4_INSTABILITY"
  delta_critical: 0.5
  weight: 1.0
  mode: "soft"
```

---

### TYPE MAPPING : Transitions qualitatives

**Définition** : Mapping direct de valeurs catégorielles vers scores pathologie.

**Métriques typiques** :
- `evolution_transition` : explosive, stable, converged...
- `evolution_trend` : increasing, decreasing, stable...

**Formule** :
```python
mapping = scoring_rule['mapping']
score = mapping.get(value, 0.5)  # Default 0.5 si non mappé
flag = score >= 0.8
```

**Configuration YAML** :
```yaml
evolution_transition:
  pathology_type: "MAPPING"
  mapping:
    explosive: 1.0
    divergent: 1.0
    stable: 0.0
    converged: 0.0
    growing: 0.3
    shrinking: 0.3
    oscillating: 0.3
  weight: 3.0
```


**Observations disponibles** :

- `evolution.{metric}.trend` : Tendance
- `evolution.{metric}.slope` : Coefficient tendance

### Conventions par défaut

Si `source` omis, extraction automatique selon préfixe :


## 12.8.6 Clarifications normatives

### 12.8.6.1 Scores = observables, pas jugements

**Les scores pathologies sont des mesures factuelles** :
- ✅ `frobenius_norm=12500 → score=0.92` (dépasse seuil 1000)
- ✅ Quantification objective d'une déviation
- ✅ Comparable entre runs

**Ils NE sont PAS** :
- ❌ Un verdict sur la qualité de Γ
- ❌ Une recommandation d'action
- ❌ Un score à optimiser

**Rôle** : Fournir données brutes pour analyse patterns (Section 12.9).

---

### 12.8.6.2 Pondération (weight) et criticité

Les `weight` dans scoring reflètent une **criticité contextuelle** :

```yaml
# Pour Γ censé préserver symétrie
asymmetry_norm:
  weight: 3.0  # Critique pour ce mécanisme

# Trace moins critique
trace_normalized:
  weight: 0.5
```

**Interprétation** :
- Poids élevé → métrique discriminante attendue pour ce Γ
- Poids faible → métrique secondaire ou exploratoire

**Impact** : Influence agrégation test_score si `mode=weighted_mean`, mais **pas le verdict final** (déterminé par patterns, Section 12.9).

---

### 12.8.6.3 Conventions numériques

**Tous les seuils** (threshold_low, threshold_high, critical_*, intervals) sont des **conventions initiales** pour exploration R0.

**Propriétés** :
- ❌ Aucune justification empirique a priori
- ❌ Aucun fondement théorique
- ✅ Baselines pour détecter déviations
- ✅ Ajustables via configs alternatives

**Calibrage** : Les patterns détectés en Section 12.9 permettent d'identifier métriques mal calibrées (toujours 0, toujours 1).

# SECTION 12.9 : VERDICTS - ANALYSE PATTERNS R0

## 12.9.1 Principe verdict R0

**Changement fondamental par rapport à versions précédentes** :

Le verdict R0 n'est **plus basé sur agrégation naïve de scores**, mais sur **analyse de patterns cross-runs**.

### Définition

> **Verdict R0** = Détection de pathologies systématiques ou de comportements contextuels via analyse de la distribution et corrélation des scores sur l'ensemble des runs.

### Verdicts autorisés

| Verdict | Signification exacte |
|---------|---------------------|
| **SURVIVES[R0]** | Aucune pathologie systématique détectée, mécanisme non absurde |
| **REJECTED[R0]** | Pathologie systématique détectée (score élevé sur ≥80% runs valides) |
| **WIP[R0-open]** | Comportement contextuel ou instable nécessitant investigation |

### Clarification terminologique

| Terme | Statut en R0 |
|-------|-------------|
| **SURVIVES[R0]** | Défini : pas de pathologie systématique |
| **Intéressant** | ❌ NON DÉFINI (jugement réservé à phases ultérieures) |
| **Prometteur** | ❌ Jugement réservé à R1+ |
| **Valide** | ❌ Jugement réservé à R2+ |

---

## 12.9.2 Patterns détectés

Le moteur d'analyse détecte **7 types de patterns** dans les scores :

### 1. NON_DISCRIMINANT

**Définition** : Métrique score toujours proche de 0 (< 0.1) sur tous runs.

**Signification** :
- Seuils trop permissifs
- Métrique non pertinente pour ce Γ
- Mécanisme systématiquement sain sur cette dimension

**Action** : Recalibrer seuils ou supprimer métrique.

**Exemple** :
```
stat_min (UNIV-001):
  - 30/30 runs: score < 0.05
  - Pattern: NON_DISCRIMINANT
  - Action: Augmenter threshold_low de 0.01 à 0.1
```

---

### 2. OVER_DISCRIMINANT

**Définition** : Métrique score toujours proche de 1 (> 0.9) sur tous runs.

**Signification** :
- Seuils trop stricts
- Pathologie généralisée (échec systématique)
- Mécanisme systématiquement pathologique sur cette dimension

**Action** : Recalibrer seuils ou confirmer pathologie systématique.

**Exemple** :
```
stat_max (UNIV-001):
  - 28/30 runs: score > 0.95
  - Pattern: OVER_DISCRIMINANT
  - Action: Augmenter threshold_high de 1000 à 5000
```

---

### 3. D_CORRELATED

**Définition** : Score varie systématiquement selon type D (variance inter-D > 0.3).

**Signification** :
- Γ spécialisé (fonctionne sur certains D, échoue sur autres)
- Comportement contextuel, pas pathologie globale

**Action** : Investiguer pourquoi certains D provoquent échec.

**Exemple** :
```
asymmetry_norm (SYM-001):
  - SYM-001: score moyen 0.85 (9/10 runs > 0.8)
  - ASY-001: score moyen 0.12 (10/10 runs < 0.2)
  - R3-001: score moyen 0.25 (10/10 runs < 0.3)
  
  Pattern: D_CORRELATED
  Interprétation: Γ diverge uniquement sur matrices symétriques
  Action: Investiguer amplification symétrie par mécanisme
```

---

### 4. MODIFIER_CORRELATED

**Définition** : Score corrélé avec modifier (proxy pour sensibilité bruit).

**Signification** :
- Γ sensible aux perturbations
- Robustesse limitée au bruit

**Action** : Tester régularisation ou pré-filtrage.

**Exemple** :
```
stat_final (UNIV-001):
  - M0 (no noise): score moyen 0.15
  - M1 (noise 0.1): score moyen 0.35
  - M3 (noise 0.3): score moyen 0.85
  - M4 (noise 0.5): score moyen 0.95
  
  Pattern: MODIFIER_CORRELATED
  Interprétation: Γ stable sans bruit, collapse avec noise ≥0.3
  Action: Tester version avec régularisation
```

---

### 5. SEED_UNSTABLE

**Définition** : Variance inter-seeds élevée (> 0.3) à D et modifier fixés.

**Signification** :
- Sensibilité initiale forte
- Instabilité stochastique

**Action** : Augmenter nombre seeds pour confirmer, ou investiguer dépendance à conditions initiales.

**Exemple** :
```
evolution_volatility (UNIV-001) sur SYM-001 + M0:
  - seed 1: score 0.25
  - seed 2: score 0.82
  - seed 3: score 0.19
  - seed 4: score 0.78
  - Variance: 0.45
  
  Pattern: SEED_UNSTABLE
  Action: Relancer avec 30 seeds au lieu de 10
```

---

### 6. SYSTEMATIC_FAILURE

**Définition** : Score élevé (> 0.7) sur ≥80% des runs sur métrique valide (ni NON_DISCRIMINANT ni OVER_DISCRIMINANT).

**Signification** :
- Pathologie systématique confirmée
- Mécanisme non viable en l'état

**Action** : Confirmer REJECTED[R0] ou investiguer cause racine.

**Exemple** :
```
frobenius_norm (UNIV-001):
  - 27/30 runs: score > 0.8
  - Pattern: SYSTEMATIC_FAILURE
  - Interprétation: Norme explose systématiquement
  - Verdict: REJECTED[R0]
```

---

### 7. CONTEXTUAL_BEHAVIOR

**Définition** : Combinaison de patterns D_CORRELATED ou MODIFIER_CORRELATED sans SYSTEMATIC_FAILURE.

**Signification** :
- Mécanisme fonctionne dans certains contextes, échoue dans autres
- Pas absurde globalement, nécessite investigation

**Action** : WIP[R0-open], investiguer domaines de validité.

---

### Étape 2 : Analyse patterns

```python
def analyze_metric_patterns(scores_data):
    """
    Détecte patterns dans scores.
    
    Returns:
        {
            'non_discriminant': list[str],
            'over_discriminant': list[str],
            'd_correlated': dict[str, dict],
            'modifier_correlated': dict[str, dict],
            'seed_unstable': list[str],
            'systematic_failures': list[str],
            'contextual_behaviors': list[str]
        }
    """
    patterns = {}
    
    # Grouper par métrique
    by_metric = group_by(scores_data, lambda x: (x['test_name'], extract_metric_key(x)))
    
    for (test_name, metric_key), metric_data in by_metric.items():
        scores = [d['score'] for d in metric_data]
        
        # 1. Non discriminant
        if max(scores) < 0.1:
            patterns.setdefault('non_discriminant', []).append(
                f"{test_name}.{metric_key}"
            )
            continue  # Skip autres analyses si non discriminant
        
        # 2. Over discriminant
        if min(scores) > 0.9:
            patterns.setdefault('over_discriminant', []).append(
                f"{test_name}.{metric_key}"
            )
            continue  # Skip autres analyses si over discriminant
        
        # Métriques valides pour analyses suivantes
        
        # 3. Corrélation D
        if has_d_correlation(metric_data):
            d_pattern = compute_d_pattern(metric_data)
            patterns.setdefault('d_correlated', {})[f"{test_name}.{metric_key}"] = d_pattern
        
        # 4. Corrélation modifier
        if has_modifier_correlation(metric_data):
            mod_pattern = compute_modifier_pattern(metric_data)
            patterns.setdefault('modifier_correlated', {})[f"{test_name}.{metric_key}"] = mod_pattern
        
        # 5. Instabilité seeds
        seed_variance = compute_seed_variance(metric_data)
        if seed_variance > 0.3:
            patterns.setdefault('seed_unstable', []).append(
                f"{test_name}.{metric_key}"
            )
        
        # 6. Échec systématique
        high_scores = sum(1 for s in scores if s > 0.7)
        if high_scores / len(scores) >= 0.8:
            patterns.setdefault('systematic_failures', []).append(
                f"{test_name}.{metric_key}"
            )
    
    return patterns
```

### Étape 3 : Décision verdict

```python
def decide_verdict_from_patterns(patterns, metric_quality):
    """
    Décide verdict depuis patterns détectés.
    
    Returns:
        (verdict: str, reason: str)
    """
    # Identifier métriques valides (discriminantes, calibrées)
    valid_metrics = [
        m for m in metric_quality 
        if metric_quality[m]['valid']
    ]
    
    # REJECTED : Pathologies systématiques sur métriques valides
    systematic = patterns.get('systematic_failures', [])
    systematic_valid = [m for m in systematic if m in valid_metrics]
    
    if len(systematic_valid) >= 3:  # Seuil: ≥3 métriques valides échouent
        return (
            "REJECTED[R0]",
            f"Pathologies systématiques sur {len(systematic_valid)} métriques valides : "
            f"{', '.join(systematic_valid[:3])}"
        )
    
    # WIP : Comportements contextuels
    has_d_context = bool(patterns.get('d_correlated'))
    has_mod_context = bool(patterns.get('modifier_correlated'))
    
    if has_d_context or has_mod_context:
        affected_count = len(patterns.get('d_correlated', {})) + \
                        len(patterns.get('modifier_correlated', {}))
        return (
            "WIP[R0-open]",
            f"Comportement contextuel détecté sur {affected_count} métriques. "
            f"Nécessite investigation ciblée."
        )
    
    # WIP : Instabilité stochastique
    if len(patterns.get('seed_unstable', [])) >= 2:
        return (
            "WIP[R0-open]",
            f"Instabilité stochastique sur {len(patterns['seed_unstable'])} métriques. "
            f"Augmenter nombre seeds pour confirmer."
        )
    
    # SURVIVES : Pas de pathologie systématique détectée
    return (
        "SURVIVES[R0]",
        "Aucune pathologie systématique détectée. "
        "Mécanisme non absurde sur espace testé."
    )
```

### Étape 4 : Génération insights

```python
def generate_actionable_insights(patterns, metric_quality):
    """
    Génère insights actionnables depuis patterns.
    
    Returns:
        list[str]: Insights formatés
    """
    insights = []
    
    # Métriques non discriminantes
    if patterns.get('non_discriminant'):
        metrics = patterns['non_discriminant']
        insights.append(
            f"⚠ Recalibrer métriques non discriminantes ({len(metrics)}) : "
            f"{', '.join(metrics[:3])}. "
            f"Action : Augmenter seuils ou supprimer métriques."
        )
    
    # Métriques over-discriminantes
    if patterns.get('over_discriminant'):
        metrics = patterns['over_discriminant']
        insights.append(
            f"⚠ Recalibrer métriques over-discriminantes ({len(metrics)}) : "
            f"{', '.join(metrics[:3])}. "
            f"Action : Augmenter seuils critiques."
        )
    
    # Corrélations D
    if patterns.get('d_correlated'):
        for metric, d_pattern in patterns['d_correlated'].items():
            failing_d = [d for d, stats in d_pattern.items() if stats['mean'] > 0.7]
            if failing_d:
                insights.append(
                    f"📊 {metric} échoue sur {', '.join(failing_d)}. "
                    f"Action : Γ spécialisé, investiguer amplification sur ces D."
                )
    
    # Corrélations modifier (bruit)
    if patterns.get('modifier_correlated'):
        metrics = list(patterns['modifier_correlated'].keys())
        insights.append(
            f"📊 Sensibilité au bruit détectée ({len(metrics)} métriques) : "
            f"{', '.join(metrics[:3])}. "
            f"Action : Tester régularisation ou pré-filtrage."
        )
    
    # Instabilité seeds
    if patterns.get('seed_unstable'):
        metrics = patterns['seed_unstable']
        insights.append(
            f"⚠ Instabilité stochastique ({len(metrics)} métriques) : "
            f"{', '.join(metrics[:3])}. "
            f"Action : Augmenter seeds (×3) ou investiguer sensibilité initiale."
        )
    
    return insights
```

```sql
CREATE TABLE GammaVerdicts (
  id INTEGER PRIMARY KEY,
  gamma_id TEXT NOT NULL,
  
  -- Configs utilisées (traçabilité pipeline)
  params_config_id TEXT NOT NULL,
  scoring_config_id TEXT NOT NULL,
  thresholds_config_id TEXT NOT NULL,
  
  -- Critères (conservés pour compatibilité/reporting)
  majority_pct REAL,                -- % configs saines (optionnel en patterns)
  robustness_pct REAL,              -- % D viables (optionnel en patterns)
  global_score REAL,                -- Score moyen (optionnel en patterns)
  
  -- Verdict
  verdict TEXT NOT NULL,            -- SURVIVES[R0] | WIP[R0-open] | REJECTED[R0]
  verdict_reason TEXT NOT NULL,
  
  -- Patterns et insights (JSON complet)
  details TEXT NOT NULL,            -- {patterns_detected, metric_quality, insights, ...}
  
  -- Metadata
  computed_at TEXT NOT NULL,
  
  UNIQUE(gamma_id, params_config_id, scoring_config_id, thresholds_config_id),
  FOREIGN KEY(params_config_id) REFERENCES ConfigRegistry(config_id),
  FOREIGN KEY(scoring_config_id) REFERENCES ConfigRegistry(config_id),
  FOREIGN KEY(thresholds_config_id) REFERENCES ConfigRegistry(config_id)
);
```
