## 3. DÉCISIONS ARCHITECTURALES CRITIQUES

### **A. Posture non gamma-centrique (Charter R0)**

❌ **INTERDIT** :
- "Ce gamma est bon/mauvais"
- "Gamma X échoue"
- Verdict global unique REJECTED/APPROVED
- Causalité gamma → outcome

✅ **AUTORISÉ** :
- "Sous gamma X, observable Y conserve propriété Z"
- "Tous gammas échouent test T → rang insuffisant ?"
- "Gamma X + modifier M3 → instabilité (interaction structurelle)"
- Profils individuels, pas hiérarchie

### **B. Séparation responsabilités (validée GPT)**

```
TestEngine :
- Calcule dynamiques (une fois, lors tests)
- Stocke événements dans observation_data
- Responsabilité : PRODUCTION faits

GammaProfiling :
- Lit événements (depuis db_results)
- Agrège, classifie, expose
- Responsabilité : INTERPRÉTATION

VerdictEngine :
- Analyses globales (variance, interactions)
- Patterns structuraux
- Responsabilité : DIAGNOSTIC SYSTÉMIQUE
```

**PRINCIPE FONDAMENTAL** : Aucun recalcul dynamique dans profiling/verdict.

## 6. RÉGIMES R0 (taxonomie validée)

### **8 régimes de base**
1. `CONSERVES_X` : Stable, faible variation (cv < 0.1)
2. `NUMERIC_INSTABILITY` : Rupture précoce + explosion (onset < 20, final > 1e20)
3. `SATURATES_HIGH` : Croissance → plateau (saturation_frac > 0.7, final > 10×initial)
4. `OSCILLATORY_UNSTABLE` : Oscillations persistantes (frac > 0.3)
5. `TRIVIAL` : Absence dynamique (cv < 0.01)
6. `DEGRADING` : Décroissance monotone (final < 0.5×initial, collapse_frac < 0.1)
7. `UNCATEGORIZED` : Aucun pattern clair
8. `MIXED::X` : Bimodalité + régime de base X

### **Événements élémentaires**
- `deviation` : Écart >10% initial
- `instability` : Rupture dérivée (P90×10)
- `oscillatory` : >10% sign changes
- `saturation` : Plateau final (std/mean < 5%)
- `collapse` : Retour brutal à ~0

### **Timelines descriptives (non causales)**
- `no_significant_dynamics`
- `early_deviation_then_plateau`
- `late_deviation_then_plateau`
- `instability_then_collapse`
- `instability_then_plateau`
- `complex_dynamics` (≥3 événements)
- `direct_saturation`
- `atypical_dynamics`

## 7. PRINCIPES MÉTHODOLOGIQUES R0

### **A. Limites explicites**
- Seuils relatifs (P90×10, cv < 0.1) : heuristiques R0, non statistiques
- Pas de distinction instabilité numérique vs structurelle (→ R2.5)
- Pas de distinction attracteur numérique vs structurel (→ R2.5)
- Timeline = chronologie pure, pas causalité

### **B. Règles de nommage**
- Descriptif > Interprétatif : "plateau" pas "recovery"
- Chronologique > Causal : "then" pas "causes"
- Observable > Jugement : "instability" pas "failure"

### **C. Validation sortie R0**
Un verdict engine R0 est correct si :
1. Suppression 30% données → profils changent structurellement
2. Retrait 1 gamma → classements tests affectés
3. Projection différente → comportements différents révélés
4. Test absurde → détecté non discriminant/mal conditionné
