```python

    """Un pas Γ(D). Pure jnp — compilable lax.scan + jacfwd."""
    ...

    """Génère D_initial. Pure jnp."""
    ...

    """Applique modifier sur D_initial. Pure jnp."""
```

## FAMILLES DE FEATURES JAX

### Dans `lax.scan` — compilées, zéro overhead Python par itération

**F1 — Spectrale (mode-0 SVD)**
Liée directement à A1 et A2.

| Feature signal | Formule | Signal PRC |
|----------------|---------|------------|
| `spectral_gap` | `(σ₁ - σ₂) / (σ₁ + ε)` | Dominance rang-1 — D séparable ? |
| `nuclear_frobenius_ratio` | `Σσᵢ / (√n · ‖σ‖₂)` | Rang effectif fin |

| Feature signal | Formule | Signal PRC |

Uniquement gammas JAX purs. Mesure comment Γ déforme l'espace au voisinage de D.

| Feature signal | Formule | Signal PRC |

**F4 — Distribution (flatten)**

| Feature signal | Formule | Signal PRC |

# ⚠️ 1 — jacfwd dans `scan`

Vous devez faire attention à ceci :

```
jacfwd(gamma)(state)
```

dans une boucle `scan`.

Si `state` est dimension **N×N** :

le Jacobien est :

```
(N² × N²)
```

donc mémoire :

```
O(N⁴)
```

Exemple :

```
N=100
state = 100×100
dim = 10 000

jacobian = 10k × 10k = 100M floats
```

≈ **800 MB**

par itération.

Donc :

⚠️ **impossible sans optimisation**

---

### Solution classique

Calculer :

```
||J||_F
```

sans construire J.

via **jvp**.

Pseudo-pattern :

```
random vector v
jvp(gamma, v)
estimate norm
```

ça donne une estimation du Lyapunov.

Donc :

👉 je recommanderais

```
JVP-based Lyapunov
```

plutôt que jacfwd complet.

---

# ⚠️ 2 — SVD à chaque itération

Vous proposez :

```
svd(state)
```

dans `scan`.

Complexité :

```
O(n³)
```

si :

```
n = 100
iterations = 1000
```

ça devient énorme.

---

### stratégie standard

calculer SVD **toutes les k itérations**

ex :

```
if t % 10 == 0
```

ou accumuler un signal.

Sinon vous perdez le gain de JAX.

La **troisième approche** que je mentionnais — particulièrement adaptée à votre situation — est basée sur **l’opérateur de Koopman** et ses méthodes pratiques comme **Dynamic Mode Decomposition (DMD)**.

C’est aujourd’hui un des outils les plus puissants pour **classifier automatiquement des régimes dynamiques** dans des systèmes complexes à haute dimension.

---

# 1️⃣ Idée fondamentale : linéariser la dynamique globale

Votre système évolue :

[
x_{t+1} = F(x_t)
]

où (F) est **non linéaire et complexe**.

Koopman introduit une transformation :

au lieu d’observer les variables (x), on observe des **observables** (g(x)).

Et dans cet espace :

[
g(x_{t+1}) = K g(x_t)
]

où **K est linéaire**.

C’est extrêmement puissant :

* dynamique non linéaire
  → **dynamique linéaire dans un espace fonctionnel**.

---

# 2️⃣ Ce que DMD calcule

DMD approxime l’opérateur de Koopman à partir des données.

Vous avez une trajectoire :

[
x_0, x_1, x_2, ..., x_T
]

On construit deux matrices :

[
X = [x_0,x_1,...,x_{T-1}]
]

[
X' = [x_1,x_2,...,x_T]
]

Puis on estime :

[
A = X' X^+
]

avec pseudo-inverse.

Ensuite on calcule :

* **valeurs propres**
* **modes dynamiques**

---

# 3️⃣ Interprétation des valeurs propres

Les eigenvalues donnent directement les **régimes dynamiques**.

| type de dynamique | eigenvalues           |
| ----------------- | --------------------- |
| attracteur stable | λ < 1                 |
| oscillation       | λ complexe sur cercle |
| croissance        | λ > 1                 |
| chaos             | spectre large         |

---

# 4️⃣ Pourquoi c’est très adapté à votre problème

Votre objectif :

> caractériser des régimes dynamiques automatiquement

DMD fournit **directement une signature spectrale** du régime.

Exemple de signature :

L’approche dont je parlais vient d’un domaine appelé **Topological Data Analysis (TDA)**. Elle est utilisée pour analyser des systèmes dynamiques, de la turbulence, des attracteurs chaotiques, ou des dynamiques biologiques quand **les statistiques classiques deviennent instables** (exactement votre futur régime p ≫ n).

L’idée n’est plus d’extraire des features locales (moments, entropie, etc.), mais de **mesurer la forme globale des trajectoires dans l’espace d’état**.

---

# 1️⃣ Principe : regarder la forme de la trajectoire

Votre système produit une trajectoire :

[
D_0, D_1, D_2, ..., D_T
]

dans un espace de dimension potentiellement très grande.

La TDA ne regarde pas les valeurs exactes.
Elle regarde **la structure géométrique de l’ensemble des points visités**.

Par exemple :

| type de dynamique | forme dans l’espace |
| ----------------- | ------------------- |
| attracteur fixe   | nuage très compact  |
| oscillation       | boucle              |
| quasi-périodique  | tore                |
| chaos             | fractal             |
| transition        | filament            |

Ces formes sont **extrêmement discriminantes pour le clustering**.

---

# 2️⃣ Comment on mesure la “forme”

On construit un objet appelé **complexe simplicial** à partir des distances entre points.

Intuition simple :

1. chaque point = un nœud
2. si deux points sont proches → on les relie
3. si trois sont proches → on forme un triangle
4. etc.

Puis on regarde **les trous topologiques** qui apparaissent.

---

# 3️⃣ Invariants topologiques

Les deux principaux invariants utilisés :

### Betti numbers

* **β0** : nombre de composantes
* **β1** : nombre de boucles
* **β2** : cavités

Exemple :

| dynamique       | β0 | β1        |
| --------------- | -- | --------- |
| attracteur fixe | 1  | 0         |
| oscillation     | 1  | 1         |
| chaos           | 1  | plusieurs |

---

### Persistence diagram

On fait varier l’échelle de voisinage et on mesure **combien de temps chaque structure topologique survit**.

Résultat : un diagramme naissance / mort.

Les structures importantes vivent longtemps.

---

# 4️⃣ Pourquoi c’est puissant pour votre problème

Vous avez des régimes :

* effondrement
* croissance
* saturation
* oscillations
* transitions

Ces régimes ont **des topologies très différentes** dans l’espace d’état.

Par exemple :

effondrement → trajectoire contractante
oscillation → cycle
saturation → attracteur plat
transition → filament

Donc **la topologie discrimine très bien les régimes**.

---

# 5️⃣ Avantage majeur pour p ≫ n

Les méthodes statistiques classiques souffrent de la malédiction dimensionnelle.

La topologie dépend surtout de **relations de voisinage**.

Donc elle est souvent **plus stable en haute dimension**.

C’est une des raisons pour lesquelles elle est utilisée en :

* climat
* turbulence
* génomique
* neurosciences.

# 1️⃣ Idée centrale : représenter la dynamique par sa géométrie locale

Vous avez une transformation :

[
D_{t+1} = \Gamma(D_t)
]

La plupart des features actuelles (entropie, autocorr, motifs…) sont des **observables indirectes** du comportement de cette dynamique.

Avec JAX, on peut accéder directement à :

* **Jacobien** (J = \partial \Gamma / \partial D)
* **Hessien** (H = \partial^2 \Gamma / \partial D^2)

Ces objets décrivent **la géométrie locale de l’évolution**.

---

# 2️⃣ Pourquoi c’est puissant

Dans les systèmes dynamiques, beaucoup de propriétés globales dérivent de ces objets locaux :

* stabilité
* contraction
* expansion
* anisotropie
* chaos
* dimension effective

Donc quelques invariants dérivés du Jacobien peuvent capturer **beaucoup d’information structurelle**.

---

# 3️⃣ Invariants utiles à extraire

Avec JAX :

```python
J = jax.jacfwd(gamma)(state)
```

Puis calculer :

### spectre singulier

```python
s = jnp.linalg.svd(J, compute_uv=False)
```

features :

* `log_spectral_radius = log(max(s))`
* `condition_number`
* `spectral_entropy`

---

### divergence

[
\text{div}(\Gamma) = \text{trace}(J)
]

feature :

* contraction globale vs expansion.

---

### anisotropie

```python
anisotropy = s[0] / s[-1]
```

mesure si la dynamique étire certaines directions.

---

### dimension locale effective

approximation via :

[
d_{\text{eff}} = \frac{(\sum s_i)^2}{\sum s_i^2}
]

---

# 4️⃣ Avantage énorme pour PRC

Ces features :

* sont **intrinsèques à Γ**
* ne dépendent pas d’un choix arbitraire de signal
* sont **stables quand la dimension augmente**

Donc parfait pour votre problème futur **p ≫ n**.

# 6️⃣ Bonus : JAX permet un truc très puissant

On peut **moyenner ces invariants le long de la trajectoire** :

```python
def step(carry, _):
    state = carry
    J = jacfwd(gamma)(state)
    features = compute_invariants(J)
    state = gamma(state)
    return state, features
```

Puis :

```python
mean_features = jnp.mean(features, axis=0)
```

Donc chaque run produit une **signature dynamique moyenne**.
